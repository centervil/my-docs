# 2026年2月22日 サイバーセキュリティニュースまとめ

## 主要ポイント
* **国内重要産業へのランサムウェア攻撃が激化：** 半導体大手アドバンテストやワシントンホテルが相次いで侵害され、世界のサプライチェーンや国内サービスに実害が及んでいます。
* **「エージェント型」マルウェアとAI武器化の進展：** 実行時にGoogle Geminiを利用して自律的に潜伏を図るマルウェア「PromptSpy」や、AIを駆使して600台超のFortiGateを侵害するキャンペーンが報告されています。
* **BeyondTrust製品の脆弱性（CVE-2026-1731）の広範な悪用：** CVSS 9.9の致命的な欠陥がランサムウェア攻撃で悪用されており、CISAは連邦機関に対し緊急のパッチ適用を命じました。

## 国内インシデントと産業リスク
日本の半導体およびサービスセクターが深刻なランサムウェア被害に直面しています。**アドバンテスト**は2月15日に侵害を検知し、一部ネットワークの隔離を余儀なくされました。同社は世界のチップ供給網における「チョークポイント（要衝）」であり、長期停止はAIや5G技術の発展に影響する恐れがあります。また、**ワシントンホテル**では、顧客データベースは無事だったものの、決済端末やチェックインシステムが停止し、現金のみの対応を迫られるなど、物理的なオペレーションに支障が出ました。

## AI技術の攻防戦
AIは攻撃と防御の両面で主役に躍り出ています。防御側では、Anthropicがコードベースをスキャンしてパッチを提案する**「Claude Code Security」**をリリースしました。一方、攻撃側では、Androidマルウェア**「PromptSpy」**がGoogle Gemini APIを介して動的にデバイス画面を分析し、ユーザーによる削除を防ぐためのステップをAIに考えさせるという、これまでにない「自律型（エージェント型）」の脅威が出現しています。

## プラットフォームの脆弱性とサプライチェーン
基盤システムや開発者ツールを狙った攻撃が加速しています。**Microsoft**は2月の月例パッチでSYSTEM特権昇格などの脆弱性を修正しました。また、AIコーディングアシスタント**「Cline CLI」**がサプライチェーン攻撃を受け、約4,000台の開発者端末に無断でAIエージェントがインストールされる事案が発生しました。さらに、**BeyondTrust**のリモートアクセス製品の脆弱性は、攻撃者がネットワーク内での横展開やデータ窃取を行うための強力な足がかりとなっています。

### 1. 半導体試験装置大手アドバンテストへのランサムウェア攻撃
2026年2月15日、アドバンテストは社内ネットワークでの異常な活動を検知し、ランサムウェア攻撃を受けたことを公表しました。同社は即座に影響範囲を隔離し、外部専門家を交えた調査を開始しました。半導体テスト分野で世界シェアを誇る同社のインシデントは、5GやAI向けチップの供給遅延を招くリスクがあり、ハイテク産業全体のサプライチェーンに対する脅威として注目されています。
[出典: Cyber Press - Ransomware Attack Disrupts Multiple Systems at Japanese Semiconductor Supplier]

### 2. 世界初のAI実行時利用マルウェア「PromptSpy」の発見
ESETの研究者は、Android端末に感染し、実行時にGoogle Gemini AIと通信するマルウェア「PromptSpy」を特定しました。このマルウェアは、AIにデバイスの状況を分析させ、最適な永続化（潜伏）手法をリアルタイムで生成させます。これにより、ユーザーがアプリを終了させたりアンインストールしたりすることを妨害する deceptive（欺瞞的）なインターフェースを動的に構築することが可能です。
[出典: The Hacker News - PromptSpy Android Malware Abuses Gemini AI to Automate Recent-Apps Persistence]

### 3. Anthropic、AI駆動の脆弱性スキャン「Claude Code Security」をリリース
AI企業のAnthropicは、ソフトウェアのコードベースをスキャンしてセキュリティ上の欠陥を発見し、修正パッチを提案する新機能「Claude Code Security」の提供を開始しました。このツールは、単なる静的解析を超えて、人間の研究者のようにデータフローを追跡し、複雑なロジックの欠陥を特定することを目指しています。攻撃者がAIを用いて脆弱性発見を自動化する中、防御側のスピードを向上させる「能動的防御」の一環と位置付けられています。
[出典: Anthropic - Anthropic Launches Claude Code Security for AI-Powered Vulnerability Scanning]

## 深掘り
### AIによる攻撃の「民主化」とFortiGate大規模侵害
2026年2月に報告されたAmazon Threat Intelligenceの調査結果は、AIが攻撃者のスキルギャップを完全に埋めている実態を浮き彫りにしました。

#### 非熟練者による大規模攻撃の成功
ロシア語圏の、技術的に未熟とされる攻撃者が、複数の商用生成AIツールを駆使することで、世界55カ国の600台以上のFortiGateデバイスを侵害することに成功しました。この攻撃では、複雑なゼロデイ脆弱性ではなく、管理ポートの露出や単要素認証（SFA）の脆弱な資格情報といった「基本的なセキュリティ不備」が狙われました。

#### 攻撃サイクルのAI自動化
AIは以下のプロセスで武器化されました：
*   **偵察ツールの開発：** 攻撃者が自力では書けない高度なスキャン用スクリプトをAIが生成。
*   **脆弱性のマッピング：** グローバル規模で設定不備のあるデバイスを高速に特定。
*   **フィッシングの洗練：** 認証情報を盗むための文面やロジックをAIで最適化。
これにより、中堅レベルの攻撃者が、従来は国家支援型グループにしか不可能だった規模の作戦を実行できる時代（攻撃の工業化）が到来しています。

### [関連知識]
#### [用語解説]
*   **PromptSpy:** LLM（大規模言語モデル）をマルウェアの「脳」として利用し、感染環境に合わせて行動を最適化する新型マルウェア。
*   **ZTNA (Zero Trust Network Access):** 「誰も信頼しない」原則に基づき、アプリ単位でアクセス制御を行う技術。アサヒグループなどがVPNからの移行を進めています。
*   **SBOM (Software Bill of Materials):** ソフトウェアの「部品表」。依存関係を透明化し、脆弱なライブラリが混入した際の影響を早期に特定するために、日本でも導入の議論が進んでいます。

#### [今後の展望]
2026年は、AIエージェントが人間のミスに代わってデータ漏洩の最大の原因になると予測されています。これに対抗するため、特権を持ったAIプロセスを監視する「Agentic Zero Trust」の導入が急務となります。また、BeyondTrustの事例が示すように、脆弱性公開から武器化までの時間が24時間を切っているため、修正パッチの適用サイクルを「数時間」単位まで短縮する運用の自動化が不可欠です。

## 結論
2026年2月下旬の情勢は、サイバー攻撃が「人対人」から「自動化されたエコシステム同士の戦い」へと完全に移行したことを示しています。アドバンテストの侵害が示すサプライチェーンの脆さと、PromptSpyに見られるAIの自律的な悪用は、従来の境界防御がもはや機能しないことを意味しています。組織は、AIを敵と見なしたガバナンスの構築と、インフラ全体の真正なゼロトラスト化を急がなければなりません。
