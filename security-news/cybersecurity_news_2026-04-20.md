# 2026年4月20日 サイバーセキュリティニュースまとめ

## 主要ポイント

* **Axios npmパッケージの深刻なサプライチェーン攻撃:** 人気JavaScriptライブラリ「Axios」のメンテナーが標的型攻撃を受け、悪意のあるバージョン（1.14.1, 0.30.4）が公開された。約3時間の公開期間にCI/CD環境へRATが混入した可能性がある。

* **AIインフラの設計上の欠陥によるRCEリスク:** Anthropicが開発した「Model Context Protocol (MCP)」に、AIエージェントと外部ツールの通信を悪用して任意コマンドを実行（RCE）できる深刻な設計上の脆弱性が発見された。

* **Adobe Acrobat/Reader ゼロデイの悪用拡大:** 既に実環境での悪用が確認されているゼロデイ脆弱性（CVE-2026-34621）について、CISAが連邦政府機関に対し4月27日までの修正を命じるなど、対応が急務となっている。

## ソフトウェアサプライチェーンの脅威

オープンソースエコシステムの信頼を揺るがす事態が発生しました。Microsoftによって「Sapphire Sleet」と特定された攻撃者が、Axiosのリードメンテナーを巧妙なソーシャルエンジニアリングで侵害し、npm認証情報を窃取しました。これにより、Windows、macOS、Linuxに対応した多段階のRATを配備する悪意のあるコードが正規パッケージに注入されました。これは、技術的保護だけでなく、開発者個人のアイデンティティ管理がサプライチェーン全体の弱点となっていることを示しています。

## AIエコシステムの構造的脆弱性

AIエージェントの普及に伴い、その通信プロトコルが新たな標的となっています。AnthropicのMCP SDKにおける設計上の欠陥は、特定のトランスポートメカニズム（STDIO）への依存に起因しており、プロンプトインジェクションを通じてサーバーの完全な制御を許す可能性があります。この脆弱性はSDKレベルに組み込まれており、下流の数千のAIアプリケーションに影響を及ぼす「AIサプライチェーンの母」とも言えるシナリオを露呈させています。

## 国内インシデントと脆弱性情勢

日本国内では、松山市営住宅の入居者情報が指定管理者のランサムウェア被害により流出したほか、オムロン製UPSの管理アプリやJuniper製OS（Junos OS）に深刻な脆弱性が確認されました。また、JIPDECの調査によれば、日本企業が身代金を支払う割合は20.1%まで減少しているものの、支払った企業の約半数が復旧に失敗しており、金銭的解決の無効性が改めて浮き彫りになっています。

# 2026年4月20日 サイバーセキュリティニュース詳細

## ニュース紹介

### 1. Axios npmパッケージ侵害：メンテナーを狙った「井戸への毒入れ」
2026年4月20日、広く利用されているJavaScriptライブラリ「Axios」の公式npmリポジトリに、悪意のあるバージョンが注入されたことが判明しました。攻撃者「Sapphire Sleet」は、魅力的な求人や協力依頼を装ってメンテナーを勧誘し、その端末にマルウェアをインストールさせることで認証情報を奪取しました。注入されたパッケージ（axios@1.14.1, axios@0.30.4）には外部依存関係「plain-crypto-js」が追加され、OSを自動判別してRATを配備する仕組みとなっていました。公開から90分以内に無効化されたものの、自動ビルドパイプラインを通じて多くの組織に影響が及んだと考えられます。
[出典: CISA - Supply Chain Compromise Impacts Axios Node Package Manager / The State of Global Cybersecurity]

### 2. Anthropic MCPにおける「設計上の」RCE脆弱性
OX Securityの研究者は、AIエージェントの通信規格であるModel Context Protocol (MCP) に、認証不要でコマンド実行を可能にする構造的な欠陥を特定しました。この脆弱性は、LLMがMCPサーバーをサブプロセスとして起動する際のSTDIO処理の不備に起因します。悪意のあるプロンプトやUIインジェクションを通じて、攻撃者はAI環境のファイル窃取やシェルアクセスを行うことができます。SDKのダウンロード数は1.5億回を超えており、プロトコル自体の設計変更が行われない限り、広範なリスクが残存し続けます。
[出典: The Hacker News - Anthropic MCP Design Vulnerability Enables RCE]

### 3. JIPDEC調査：日本企業のランサムウェア対応の変化
2026年4月20日に発表されたJIPDECの調査結果により、日本企業のランサムウェアに対する姿勢の変化が明らかになりました。身代金を支払う企業の割合は2024年の26.9%から、2026年には20.1%まで低下しています。特筆すべきは、身代金を支払った139社のうち、システム復旧に成功しなかった割合が非常に高く、一方で自力で復旧できた企業も同数程度存在した点です。復旧には「1週間から1ヶ月」を要するケースが最も多く、事業継続計画（BCP）の重要性がデータで証明されました。
[出典: ITmedia NEWS - ランサムウェア被害を受けても「身代金は支払わない」企業の割合が年々増加 / JIPDEC調査]

## 深掘り

### 物理的レイヤーへの脅威：光ファイバーのマイクロフォン化
ネットワークインフラそのものを物理的な盗聴器に変える新しい攻撃手法が注目されています。

#### 光ファイバー盗聴のメカニズム
研究によれば、インターネットを支える光ファイバーケーブルを「音響センサー」として悪用することが可能です。これは「レイリー後方散乱」という物理現象を利用したもので、ケーブル付近の会話による微細な振動を光源側で検出し、音声を復元します。

#### セキュリティ上の影響
この手法は、ネットワーク暗号化や物理的なアクセス制御を迂回します。ケーブル自体がセンサーとなるため、政府機関やデータセンターの周辺にある標準的な通信インフラが、そのまま大規模な監視ネットワークへと変貌するリスクを孕んでいます。

### 関連知識

#### 用語解説
*   **Sapphire Sleet:** Microsoftが追跡している、主に開発者や技術者を標的にソーシャルエンジニアリングを用いる高度な脅威アクター。
*   **MCP (Model Context Protocol):** AIモデルが外部データやツールと安全に（とされる）通信を行うためのオープンプロトコル。
*   **プロトタイプ汚染 (Prototype Pollution):** JavaScriptなどの動的言語において、オブジェクトの基本構造を書き換えることで任意コード実行を狙う脆弱性（AdobeのCVE-2026-34621に該当）。

#### 今後の展望
AxiosやMCPの事例が示すように、サイバーセキュリティの境界線は「コードのバグ」から「プロトコルのロジック」や「人間という脆弱性」へと移行しています。今後は、静的なスキャンだけでなく、アイデンティティ（OIDCフローの採用）やAIエージェントの動的な振る舞いを監視する「アイデンティティ中心型」および「アーキテクチャ認識型」の防御姿勢が必要不可欠となります。

## 結論
2026年4月21日現在の脅威環境は、サプライチェーン、AIプロトコル、そして人間の信頼という3つのレイヤーが同時に標的となっています。Axiosの侵害は信頼されたメンテナーが攻撃の起点となり得ることを証明し、MCPの脆弱性はAIの急速な普及の陰に潜む設計リスクを浮き彫りにしました。日本企業においては、ランサムウェアへの金銭的支払いの無効性が認識されつつあるものの、UPSやネットワーク機器の脆弱性が産業基盤を脅かしています。組織は技術的なパッチ適用を超え、信頼とアイデンティティの再定義を含む、抜本的なレジリエンスの再構築が求められています。
