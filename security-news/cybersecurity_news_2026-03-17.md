# 2026年3月17日 サイバーセキュリティニュースまとめ

## 主要ポイント

* **ソフトウェアサプライチェーン攻撃の高度化（GlassWorm）:** GitHubトークンを窃取し、Gitの履歴（rebase/force-push）を書き換えることで、プルリクエストの痕跡を残さずマルウェアを注入する手口が確認されました。
* **国際的なボットネット「SocksEscort」の解体:** 163カ国、約36万9,000件のIPアドレスを悪用していた犯罪用プロキシサービスが、法執行機関の共同作戦「Operation Lightning」により制圧されました。
* **次世代IT戦略と「協調防御」への転換:** 大阪で開催された「CIO Japan Summit 2026」において、AI駆動の脅威に対し、組織単体ではなく業界・パートナー間で連携する「協調防御」の重要性が議論されました。

## ソフトウェアサプライチェーンと開発環境の脅威
開発者のローカル環境やIDE（VS Code等）を起点とする攻撃が深刻化しています。**GlassWormキャンペーン**では、悪質な拡張機能を通じてGitHubトークンを盗み出し、Pythonリポジトリのソースコードにマルウェアを直接注入します。また、開発者がターミナルにコマンドを貼り付ける習慣を悪用した「ClickFix（InstallFix）」手法により、AIツールを装ってインフォスティーラーを感染させる事例も急増しています。

## ランサムウェア戦術の進化と国際的な法執行
ランサムウェア運営者は、初期侵入をブローカーに頼るモデルから、ウェブサイト改ざんを用いたソーシャルエンジニアリング（ClickFix）など、自前でアクセスを確保する手法へシフトしています。一方で、法執行機関はボットネット「SocksEscort」のサーバーやドメインを差し押さえ、350万ドルの暗号資産を凍結するなど、犯罪インフラの解体を加速させています。

## 国内インシデントと重要インフラの脆弱性
日本国内では、**中央学院大学**がVPN経由の侵入によるランサムウェア被害の調査結果を公表し、学生らの初期設定パスワード等が流出した可能性を認めました。また、**トンボ飲料**はRDPソフトが侵入経路となったことを最終報告しています。脆弱性関連では、Microsoft Web Deploy 4.0のRCE（遠隔コード実行）エクスプロイトが公開されたほか、CISAがFileZenの脆弱性を「悪用済みリスト（KEV）」に追加し、緊急の対応を求めています。

# 2026年3月17日 サイバーセキュリティニュース詳細

## ニュース紹介

### 1. GlassWorm攻撃：GitHub履歴を書き換えてPythonリポジトリを汚染
脅威アクターは、盗んだGitHubトークンを使用してリポジトリの履歴を「rebase」および「force-push」することで、悪意あるコードを注入しています。この手法は、元のコミットのメッセージや著者を保持したままコードを書き換えるため、GitHubのUI上にプルリクエストの痕跡が残らず、検知が極めて困難です。影響はDjangoアプリや機械学習リサーチコードなど、150以上のリポジトリに及んでいます。
[出典: The Hacker News - GlassWorm Attack Uses Stolen GitHub Tokens to Force-Push Malware Into Python Repos]

### 2. 国際共同捜査によりプロキシボットネット「SocksEscort」を解体
米司法省やEuropolなどは、家庭用ルーター等をボットネット化していた「SocksEscort」を制圧しました。このサービスは、感染したデバイスを通じてサイバー犯罪者の通信を中継し、位置情報を隠蔽するために利用されていました。マルウェア「AVrecon」は、ルーターのファームウェアを書き換えて永続性を確保し、ユーザーによる初期化やアップデートを無効化する極めて執拗な性質を持っています。
[出典: The Hacker News - Authorities Disrupt SocksEscort Proxy Botnet Exploiting 369,000 IPs Across 163 Countries]

### 3. Microsoft Web Deploy 4.0 に任意のコード実行の脆弱性、エクスプロイトが公開
2025年8月に修正されたMicrosoft Web Deploy 4.0のシリアライズデータ処理の不備（CVE-2025-53772）について、実際に遠隔からコードを実行可能なエクスプロイトコードが公開されました。認証済みアカウントが必要ですが、IISの権限で動作するため、SYSTEM権限の奪取に繋がるリスクが高いと分析されています。未パッチの環境に対する攻撃の激化が懸念されます。
[出典: ScanNetSecurity - Microsoft Web Deploy 4.0 において遠隔からの任意のコード実行につながるシリアライズデータ処理の不備]

## 深掘り

### 信頼の基盤を揺るがす「Git履歴の操作」と開発環境の汚染

2026年3月の動向において最も注目すべきは、**「開発プロセスそのものへの信頼」を逆手に取った攻撃**の成熟です。

#### Git操作の武器化
GlassWormキャンペーンが用いる「force-push」による履歴の書き換えは、従来の「不正なコミットを追加する」攻撃とは一線を画します。Gitの整合性を保ちつつ既存の正常なファイルを汚染するため、開発者がコードレビューを済ませたはずのソースコードが、いつの間にかマルウェアに入れ替わっているという事態を引き起こします。これは、ソースコード管理システム（SCM）のUIに頼った監査の限界を示しています。

#### 協調防御（Coordinated Defense）へのシフト
「CIO Japan Summit 2026」で議論された通り、AIによって攻撃が「工業化」され、一組織の防御を容易に突破する現在、**「業界全体での脅威情報の共有」と「レジリエンス（回復力）」**が不可欠です。中央学院大学の例に見られるように、復旧には多大な時間を要するため、侵入を防ぐ「守るIT」だけでなく、事業を継続させる「支えるIT」の両立が経営課題となっています。

### [関連知識]

#### 用語解説
*   **ClickFix:** 偽のブラウザエラーやCAPTCHAを表示し、解決のために特定のコマンドをコピーしてターミナルやPowerShellに貼り付けさせるソーシャルエンジニアリング手法。
*   **AVrecon:** C言語で書かれ、MIPSやARMアーキテクチャ（ルーター等）を標的とするマルウェア。カスタムファームウェアをフラッシュすることで永続性を確保する。
*   **CIRCIA:** 米国の2022年重要インフラサイバーインシデント報告法。2026年3月17日・18日にかけて、主要セクターでの報告義務化に関する重要な節目を迎えました。

#### 今後の展望
2026年4月1日からは、日本国内においてIPAの「SECURITY ACTION」管理システムがGビズIDと連携し、企業のセキュリティ対策状況のデジタル管理が強化されます。今後は、GlassWormのような巧妙なサプライチェーン攻撃に対抗するため、コードの署名検証や、IDE拡張機能の厳格な権限管理、そして開発履歴の異常を検知する動的な監視体制の構築が必須となるでしょう。

## 結論
2026年3月中旬の情勢は、**「信頼されている正規のツールやプロセス（Git、リモートデスクトップ、管理ツール）」の悪用**が攻撃の主流となったことを示しています。中央学院大学やトンボ飲料の事例は、VPNやRDPといった「境界の扉」の管理不備が致命的な結果を招くことを再認識させました。組織は「侵害されることを前提」とし、AIや外部パートナーと連携した**「協調防御」と「自律的なガバナンス」**へアーキテクチャを刷新することが求められています。
