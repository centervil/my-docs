# 2026年5月2日 サイバーセキュリティニュースまとめ

## 主要ポイント

*   **大手セキュリティベンダーを狙った戦略的侵害:** Trellix社がソースコードリポジトリへの不正アクセスを確認しました。防御側の内部ロジック流出は、将来的な検知回避やゼロデイ脆弱性発見に繋がるリスクを孕んでいます。
*   **アイデンティティ中心の「高速度」SaaS恐喝:** Snarky Spider等の攻撃グループが、ボイスフィッシング（vishing）とSSO（シングルサインオン）の悪用を組み合わせ、侵入からわずか1時間以内にデータ窃取を開始する極めて迅速な攻撃を展開しています。
*   **信頼されたプラットフォームの武器化と「工業化」されたフィッシング:** Google AppSheetを悪用して3万件のFacebookアカウントを窃取した「AccountDumpling」キャンペーンなど、正規サービスを隠れ蓑にした攻撃が工業規模で実行されています。

## 脆弱性とインフラの防衛

CISAは、D-Link製ルーターやSimpleHelp、Samsung MagicINFO 9に存在する4つの脆弱性を「悪用確認済み（KEV）」カタログに追加し、2026年5月までの修正を命じました。また、Windows Shellの脆弱性（CVE-2026-32202）がロシア背景のAPT28によってアクティブに悪用されているほか、Linuxカーネルでは「Copy Fail」と呼ばれる深刻な権限昇格の欠陥が報告されています。

## 日本国内のインシデント情勢

国内では、日本郵船（NYK）の船舶燃料調達システムやアルプスアルパインのVPNシステムへの不正アクセス、イビデンのウェブサイト改ざんなど、重要インフラおよび製造業を狙った侵害が相次いでいます。また、IPAは「10大脅威 2026」において、AIの悪用に伴うリスクを初めて主要な脅威として選出しました。

## ソフトウェアサプライチェーンの脅威

開発環境への「シフトレフト」攻撃が深刻化しています。CAMPFIREのGitHubアカウント侵害に加え、人気Pythonフレームワーク「PyTorch Lightning」の汚染パッケージが発見されるなど、開発者が信頼するリポジトリを起点とした認証情報の窃取が組織的に行われています。

# 2026年5月2日 サイバーセキュリティニュース詳細

## ニュース紹介

### 1. Trellix：ソースコードリポジトリへの不正アクセスを確認
セキュリティ企業のTrellixは、内部ソースコードリポジトリの「一部」に対する不正アクセスが発生したことを公表しました。現在のところ、製品の配布プロセスへの影響や野生下での悪用は確認されていませんが、EDR/XDRの検知ロジックが攻撃者に露出したことで、将来的に高度な検知回避技術に悪用される恐れがあります。
[出典: The Hacker News - Trellix Confirms Source Code Breach With Unauthorized Repository Access]

### 2. ベトナム背景の「AccountDumpling」：Google AppSheetを悪用し3万件のアカウントを侵害
ベトナムを拠点とする攻撃グループが、Google AppSheetを「フィッシングリレー」として悪用し、約3万件のFacebookビジネスアカウントを侵害しました。正規のドメイン（appsheet.com）から送信されるため、スパムフィルターを容易にバイパスし、Metaサポートを騙る偽ページへ誘導して認証情報と2要素認証コードをリアルタイムで窃取します。
[出典: The Hacker News - 30,000 Facebook Accounts Hacked via Google AppSheet Phishing Campaign]

### 3. Windows Shellの脆弱性（CVE-2026-32202）をAPT28が悪用
Microsoftは、Windows Shellに存在するなりすまし脆弱性が、ロシアの国家背景アクターAPT28（Fancy Bear）によって悪用されていることを認めました。この攻撃は、細工されたショートカット（LNK）ファイルを用いる「ゼロクリック」の資格情報窃取ベクトルであり、未修正の認証強制の欠陥を突いて、標的の端末を攻撃者のサーバーへ強制的に認証させます。
[出典: The Hacker News - Microsoft Confirms Active Exploitation of Windows Shell CVE-2026-32202]

## 深掘り

### SaaS恐喝と「アイデンティティ・ダークマター」の脅威

現代の攻撃者は、ネットワークの境界を壊すのではなく、**盗んだ権限で「ログイン」する戦術**を極限まで洗練させています。

#### 高速度なSaaS恐喝（Cordial Spider / Snarky Spider）
Cordial SpiderやSnarky Spiderといったグループは、ITヘルプデスクを装った**ボイスフィッシング（vishing）**を用いて従業員を騙し、SSO（シングルサインオン）の認証情報を奪取します。特筆すべきは、侵入から**1時間以内**にデータの流出を開始するその速度です。彼らは新しいデバイスをIdPに登録し、通知を削除するルールを設定することで、防御側に気づかれる前にSaaSエコシステム全体を横断的に侵害します。

#### セキュリティ基盤の「信頼」の武器化
Trellixの事例や、GitHub、PyPIなどのサプライチェーン攻撃が示す通り、攻撃の焦点は「防御ツール」や「管理インフラ」そのものの掌握に移っています。一箇所の侵害が、そのツールを信頼して利用する数万の顧客や開発者へドミノ倒しのように波及する「システム的リスク」が顕在化しています。

### [関連知識]

#### 用語解説
*   **AiTM (Adversary-in-the-Middle):** 攻撃者がユーザーと正規サイトの間に介在し、認証トークンやMFAコードをリアルタイムで横取りする手法。
*   **CPN (Covert Proxy Network):** 侵害された家庭用ルーター等を経由し、攻撃トラフィックを正規の居住者通信に紛れ込ませて隠蔽する分散型ネットワーク。
*   **KEV (Known Exploited Vulnerabilities):** CISAが管理する、実際に悪用が確認された脆弱性のカタログ。連邦政府機関には期限内のパッチ適用が義務付けられます。

#### 今後の展望
今後は「パッチ管理」のみならず、アイデンティティ管理において**FIDO2**や**MFAのハードニング**といった「フィッシング耐性」を持つ認証への強制的な移行が必要です。また、IPAが指摘するように、AIを悪用した攻撃の高度化に対抗するため、防御側もAIを統合した自律型防御（AI駆動型SOC）への投資が企業の生存を分ける鍵となるでしょう。

## 結論

2026年5月初頭の情勢は、**「システムの信頼（サプライチェーン）」と「人間の心理（ヴィッシング）」の二重の脆弱性**が、AIとクラウドの力で工業化されている現状を浮き彫りにしました。セキュリティベンダーのソースコード流出やSaaS環境での高速度な恐喝は、もはや従来の境界防御では対応不可能な領域に脅威が到達したことを意味しています。組織は「侵害を前提」とし、アイデンティティの保護とサプライチェーン全体の可視化を最優先課題に据えるべきです。
