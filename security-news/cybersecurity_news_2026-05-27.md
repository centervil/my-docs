# 2026年5月27日 サイバーセキュリティニュースまとめ

## 主要ポイント

* **GitHub Enterprise Serverにおけるクリティカルな脆弱性の修正**: SSRF（サーバサイドリクエストフォージェリ）の脆弱性（CVE-2026-9312）やパッケージ署名キーの失効など、開発インフラの根幹を揺るがす欠陥が判明し、即時の対応が求められています。

* **AIによる脆弱性発見の加速と運用のボトルネック**: AnthropicのAI「Claude Mythos」が1ヶ月で1万件を超える重大な脆弱性を特定しましたが、人間による修正パッチの検証や適用作業がその速度に追いつかないという新たな課題が浮き彫りになっています。

* **ランサムウェア経済の「非対称性」と手法の変容**: 攻撃者の侵入コスト約7万円に対し、企業の復旧コストは約2.3億円という「3,500倍」の格差が攻撃のビジネス化を支えています。また、暗号化を行わずデータの窃取と漏洩脅迫に特化する「データ恐喝型」への移行が鮮明になっています。

## 深刻な脆弱性とインフラ保護

開発プラットフォームやセキュリティ製品自体の脆弱性が、攻撃の足がかりとして狙われています。GitHub Enterprise Server (GHES) では認証なしで機密情報を取得される恐れがある欠陥が修正され、署名キーのローテーションという異例の対応も行われました。また、本来防御の要であるTrendAI Apex Oneが実環境で悪用され、管理サーバーを通じて悪意あるコードを全端末へ配布可能になるという「信頼の悪用」が深刻な脅威となっています。

## 産業制御システム（ICS/OT）のレジリエンス

製造現場のダウンタイムの約80%は、制御システムへの直接攻撃ではなく、依存する管理用ITシステムの障害に起因する「間接的なOT停止」です。精電舎電子工業の事例のように、サイバー攻撃の疑いでネットワークを遮断せざるを得ない状況は、企業の物理的な操業停止に直結します。デジタル機能の喪失を前提とした、手動操作やアナログな手段による「縮退運用」の設計が、真のレジリエンスとして重要視されています。

## AI駆動の脅威と防御の攻防

AIは脆弱性管理に劇的な変化をもたらしています。「Project Glasswing」による大規模な自動監査は防御に大きく貢献する一方、攻撃者も同様のAIを用いてパッチ公開前に攻撃を仕掛ける「ゼロデイ攻撃」の自動化を進めています。Microsoft Defenderの権限昇格脆弱性が実環境で悪用された事例に見られるように、攻撃者は高権限を持つ防御ツール自体の欠陥を突くことで、静かに、かつ確実に組織内でのラテラルムーブメント（横展開）を実現しています。

## ニュース紹介

### 1. GitHub Enterprise Serverに複数のクリティカルな脆弱性

GitHubは、オンプレミス版の開発プラットフォーム「GitHub Enterprise Server (GHES)」に複数の深刻な脆弱性が存在することを公表しました。特にCVE-2026-9312（SSRF）は、入力検証の不備を突くことで、攻撃者が認証を必要とせずに内部サービスへリクエストを送信し、機密性の高い認証情報を窃取できる恐れがあります。さらに、リリースパッケージの署名キーが失効したため、パッチ適用前に公開キーのローテーションを行う必要があるという、インフラ全体に影響を及ぼす事態となっています。

[出典: Security NEXT - 「GitHub Enterprise Server」にクリティカル脆弱性 - 修正版が公開](https://www.security-next.com/185038)

### 2. セキュリティ製品「TrendAI Apex One」を標的とした実悪用攻撃

トレンドマイクロの企業向けセキュリティ製品「TrendAI Apex One」および「Vision One Endpoint Security」に、複数の深刻な脆弱性が判明しました。中でもオンプレミス版に存在するパストラバーサルの脆弱性（CVE-2026-34926）については、実環境での悪用が既に確認されています。この脆弱性が悪用されると、認証済みの攻撃者がサーバー上のファイルを改ざんし、本来は保護の対象であるセキュリティエージェントに対して悪意のあるコードを「正当な更新」として配布できる危険性があります。

[出典: JPCERT/CC - [Updated] Alert Regarding Multiple Vulnerabilities in Trend Micro Products Including TrendAI Apex One](https://www.jpcert.or.jp/english/at/2026/at260014.html)

### 3. AI「Claude Mythos」による脆弱性発見の津波と運用の限界

Anthropicが主導する「Project Glasswing」は、AIエンジン「Claude Mythos」を活用し、わずか1ヶ月で1万件以上の重大な脆弱性を発見するという驚異的な成果を上げました。しかし、この大量検出が新たな問題を引き起こしています。AIによる発見の速度に対し、人間が行う「脆弱性の真偽確認」「修正パッチの開発」「既存システムへの影響調査」といった後続の運用プロセスが追いつかず、防御側に巨大な未処理リスト（バックログ）が発生しています。

[出典: note - 5/27、今日のセキュリティニュース｜かーでぃ / KAMADA Makoto](https://note.com/kerdy/n/na73f205d03e5)

## 深掘り

### IT障害が招く「間接的なOT停止」という構造的リスク

現代の製造業や重要インフラにおいて、物理的な生産ライン（OT）は、バックエンドのITシステム（在庫管理、物流、勤怠管理、認証基盤など）と密接に統合されています。このことが、サイバー攻撃の影響範囲を劇的に拡大させています。

#### 80%のダウンタイムはIT起因
ICS Security Conference 2026の報告によれば、OT環境のダウンタイムの約80%は、制御機器そのものの侵害ではなく、依存するITシステムの障害による「間接的なOT停止」です。例えば、精電舎電子工業の事例では、システム障害を受けて感染拡大を防ぐために全ネットワークを遮断した結果、業務が停止するという、防御的な措置が操業停止を招くジレンマが発生しています。

#### デジタル喪失を見据えた「縮退運用」の必要性
IT環境が侵害されても物理的な操業を継続するためには、設計段階から「デジタル機能が失われた状態」を想定する必要があります。過去の福島第一原発事故や海外メーカーの事例では、紙の記録、人間の五感、手動操作に頼ることで、デジタル制御なしに事業を継続した例が紹介されています。MVA（最小実行アーキテクチャ）やMVP（最小実行プロセス）の概念に基づき、アナログな継続手段をあらかじめ準備しておくことが、AI時代の高速な攻撃に対する最後の防衛線となります。

### 関連知識

#### 用語解説
* **SSRF (Server-Side Request Forgery)**: サーバーが外部や内部の他のシステムへリクエストを送信する機能を悪用し、本来アクセスできないリソースへの攻撃を行う手法。GHESの事案で注目されました。
* **データ恐喝型 (Data Extortion)**: データの暗号化を行わず、窃取した情報の公開を盾に身代金を要求する手法。バックアップによる復旧を無効化し、企業のブランド価値を人質に取ります。
* **Project Glasswing**: Anthropicが主導する、AIを活用してソフトウェアの脆弱性を自律的に発見・修正するプロジェクト。その心臓部であるAIモデルが「Mythos（ミュトス）」です。

#### 今後の展望
AIによる「攻撃の高速化・大量化」に対抗するためには、防御側もAIを統合した自動パッチ検証や、アイデンティティ管理（IAM）のリアルタイム監視を導入せざるを得ません。一方で、日本政府によるSCS評価制度の開始やJC-STARの導入、サプライチェーン全体のセキュリティ・ベースラインの底上げなど、政策面での強制力を持った対策が本格化すると予測されます。

## 結論

2026年5月のセキュリティ環境は、AIによる脆弱性発見能力の爆発的向上と、それによって露呈した人間主導のパッチ管理プロセスの限界を象徴しています。ランサムウェアが極めて高い投資対効果を持つ「ビジネス」として定着し、セキュリティ製品や開発基盤そのものが攻撃の道具に転じている現状では、「侵入を防ぐ」だけでなく「侵害されても事業を止めない（縮退運用）」ためのレジリエンス構築が、企業の存続を左右する決定的な要素となっています。
