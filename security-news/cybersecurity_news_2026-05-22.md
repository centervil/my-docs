# 2026年5月22日 サイバーセキュリティニュースまとめ

## 主要ポイント

* **国際的な法執行機関による犯罪インフラの相次ぐ解体**: 25以上のランサムウェアグループが利用していた「First VPN」の解体と、200万台規模の「Kimwolf」ボットネット運営者の逮捕が発表されました。

* **基盤ソフトウェアを狙った「負のタイム・トゥ・エクスプロイト」の加速**: TrendAI Apex One や Microsoft Defender、Drupal などのエンタープライズ製品でゼロデイ脆弱性の実悪用が相次いで確認されています。

* **国内重要インフラおよび公的機関での深刻な事案発生**: 日本医科大学武蔵小杉病院でのランサムウェアによるシステム停止や、静岡県警による1万4000人分の個人情報誤公開が明らかになりました。

## 国際共同捜査によるインフラ解体

欧米の当局は、サイバー犯罪者が匿名化や偵察に利用していた「First VPN」サービスを解体し、33台のサーバーを差し押さえるとともに、ウクライナで管理者を逮捕しました。同時に、世界で約200万台のデバイスを制御し記録的なDDoS攻撃に関与したボットネット「Kimwolf」の運営者がカナダで逮捕され、45の攻撃代行プラットフォームが閉鎖されました。これらの作戦は、攻撃者の機動力を根本から削ぐ「上流での脅威抑制」を目的としています。

## 基盤ソフトウェアのゼロデイ脆弱性

エンタープライズ向けのエンドポイント保護や管理ツールを標的とした攻撃が激化しています。TrendAI Apex One のディレクトリトラバーサル脆弱性（CVE-2026-34926）や、Microsoft Defender の権限昇格およびDoS脆弱性が実環境で悪用されており、CISAは即時のパッチ適用を命じています。また、Drupal の PostgreSQL API における SQL インジェクション脆弱性では、公開から数時間で1万5000件以上の攻撃試行が観測されるなど、AIを活用したと見られる極めて高速な武器化が進んでいます。

## 国内の重要インフラおよび行政インシデント

国内では、日本医科大学武蔵小杉病院がランサムウェア攻撃を受け、ナースコールシステムの停止や約1万人の個人情報流出が発生しました。侵入経路は保守用VPNの脆弱なパスワードであったとされています。また、静岡県警では公文書開示の際のマスキング不備により、約1万4000人分の個人情報が判読可能な状態で交付されるという人的ミスに起因する漏洩が発生しました。

# 2026年5月22日 サイバーセキュリティニュース詳細

## ニュース紹介

### 1. 日本医科大学武蔵小杉病院におけるランサムウェア被害と1万件の情報漏洩

2026年5月22日、日本医科大学武蔵小杉病院は記者会見を開き、ランサムウェア攻撃によって患者約1万人分の氏名や住所が流出したと発表しました。5月9日にナースコールシステムの不具合で発覚し、サーバーには1億ドルを要求するメッセージが残されていました。調査の結果、外部メーカーが保守に利用していたVPNゲートウェイのパスワードが脆弱で、総当たり攻撃（ブルートフォース）によって突破されたことが原因と特定されました。

[出典: 日本医科大学武蔵小杉病院 - 【日本医科大学武蔵小杉病院】サイバー攻撃で患者約1万人分の個人情報漏えい](https://www.youtube.com/watch?v=RRuNeeCgKUs)

### 2. TrendAI Apex One における実悪用済みのゼロデイ脆弱性 (CVE-2026-34926)

TrendAI（トレンドマイクロ）は、オンプレミス版の Apex One に存在するディレクトリトラバーサルの脆弱性 CVE-2026-34926 を修正しました。この脆弱性は、認証済みのローカル攻撃者がサーバー上の設定テーブルを書き換えることで、管理下のセキュリティエージェントに悪意のあるコードを配布することを可能にします。既に実環境での悪用が確認されており、CISAの KEV（既知の悪用された脆弱性）カタログに追加されています。

[出典: JPCERT/CC - JVNVU#90583059: トレンドマイクロ製企業向けエンドポイントセキュリティ製品における複数の脆弱性（2026年5月）](https://jvn.jp/vu/JVNVU#90583059/)

### 3. TanStack サプライチェーン攻撃に起因する Grafana Labs のソースコード窃取

Grafana Labs は、同社の GitHub リポジトリからソースコードが不正に持ち出されたことを明らかにしました。このインシデントは、5月11日に発生した TanStack 等のサプライチェーン攻撃の際に漏洩した開発者トークンが、適切にローテーション（更新）されていなかったことが直接の原因です。顧客データへの影響はないとしていますが、内部ソースコードの流出により、将来的に新たな脆弱性が発見・悪用される二次的なリスクが懸念されています。

[出典: SecurityWeek - Grafana Says Codebase and Other Data Stolen via TanStack Supply Chain Attack](https://www.securityweek.com/grafana-says-codebase-and-other-data-stolen-via-tanstack-supply-chain-attack/)

## 深掘り

### サプライチェーンにおける「信頼」の管理とガバナンス

直近のインシデントは、組織が「信頼」している外部パートナーや開発ツールが最大の弱点となっていることを示しています。

#### 非人間アイデンティティ (NHI) の放置リスク
Grafana の事例は、パートナー企業の侵害が自社の侵害に直結する「連鎖型」の脅威を象徴しています。APIキーやトークンといった「非人間アイデンティティ」の管理不足（未更新）が、境界防御を無意味にする侵入経路を提供してしまいました。

#### 日本独自の規制対応：SCS評価制度の始動
経済産業省が策定した「SCS評価制度」は、2026年度末の運用開始に向けて詳細が固まりました。★3から★5の格付けにより、サプライチェーン全体のセキュリティ対策を「見える化」し、取引条件として一定のレベルを求めることで、日本企業全体の底上げを図る狙いがあります。

### 関連知識

#### 用語解説
* **Bulletproof VPN**: 犯罪利用を目的とし、法執行機関からの要請に応じないことを売りとする秘匿性の高い通信サービス。
* **SCS評価制度 (Supply Chain Security Evaluation System)**: 経済産業省が主導する、サプライチェーン強化のためのセキュリティ対策評価制度。
* **ディレクトリトラバーサル**: 入力値の検証不備を突き、意図しないディレクトリやファイルにアクセスする攻撃手法。

#### 今後の展望
脆弱性の公開から攻撃までの「ウィンドウ」は AI によって数時間単位に短縮されており、人間の手動パッチ管理は限界に達しつつあります。今後は、WAF による仮想パッチや AI エージェントによる自律的な防御、および SCS評価制度に準拠した委託先管理の自動化が不可欠となるでしょう。

## 結論

2026年5月現在の脅威環境は、攻撃者が「信頼された経路（VPN、開発トークン、セキュリティソフト）」を優先的に突くフェーズにあります。日本医科大学や Grafana の事例は、従来の境界型防御の脆弱性を浮き彫りにしました。組織は、ID管理の徹底と、間もなく本格始動する SCS評価制度を軸とした、サプライチェーン全体のレジリエンス構築を急ぐ必要があります。
