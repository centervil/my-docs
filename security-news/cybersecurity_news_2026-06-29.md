# 2026年6月29日 サイバーセキュリティニュースまとめ

## 主要ポイント

* **Linuxカーネルの深刻な脆弱性「DirtyClone」の詳細とPoCが公開**: ローカルユーザーがroot権限を奪取可能な脆弱性（CVE-2026-43503）の技術詳細が公表されました。マルチテナントのクラウド環境やコンテナ環境に高いリスクをもたらします。

* **自律型AIエージェント「Claude Code」を標的とした間接プロンプトインジェクション**: 悪意のあるリポジトリを読み込ませることで、開発者のマシン上でリバースシェルを起動させる攻撃手法が実証されました。AIへの信頼を逆手に取った高度な手口です。

* **KDDI共有メール基盤の侵害、パスワード強制リセットの段階的実施へ**: 最大1,422万件の認証情報が流出した恐れがある事案で、BIGLOBEやピカラ（STNet）などのISP各社が、7月1日より順次強制リセットを実施すると発表しました。

## インフラ基盤とサプライチェーンの構造的リスク

国内の通信インフラにおいて、**共有プラットフォームの脆弱性が広範囲に被害を及ぼす「デジタルサプライチェーン・コンタジョン（感染）」**が深刻化しています。KDDIが提供するメール管理システムの脆弱性悪用により、BIGLOBE、@nifty、J:COM、ピカラ、コミュファ光、CPIの利用者データが露出しました。また、日産自動車はOracle PeopleSoftのゼロデイ脆弱性（CVE-2026-35273）を突いた攻撃グループ「ShinyHunters」により、従業員データが流出したことを確認しました。

## AIと次世代の技術的脅威

AI技術は、攻撃側が「偵察からエクスプロイト実行まで」を自律化させる手段として進化しています。**「Claude Code」への攻撃**は、リポジトリ内に悪意あるコードを含めず、AIのトラブルシューティング機能と外部のDNSインフラを組み合わせることで、従来のセキュリティ検知を完全に回避します。一方、防御側でもIPAが「AIの利用をめぐるサイバーリスク」を組織の10大脅威の3位に初選出するなど、シャドーAIやハルシネーションへの警戒を強めています。

## 国家支援型アクターと国際法執行

ロシア情報機関に関連する攻撃グループ（UNC5792, UNC4221）が、**SignalやWhatsAppの「バックアップ復元キー」を窃取**するソーシャルエンジニアリングを強化しており、米政府は最大1,000万ドルの懸賞金を提示しました。また、法執行機関は2026年FIFAワールドカップの不正ストリーミングに使用された約400のドメインを押収し、情報の窃取やマルウェア拡散の温床となっていたインフラの解体を進めています。

# 2026年6月29日 サイバーセキュリティニュース詳細

## ニュース紹介

### 1. Linuxカーネルの特権昇格脆弱性「DirtyClone」 (CVE-2026-43503)
JFrogの研究チームは、Linuxカーネルのネットワークスタックに存在する深刻な脆弱性「DirtyClone」の実証コード（PoC）を公開しました。この脆弱性は、ソケットバッファが共有ページキャッシュメモリを参照する際の不備に起因し、IPsecの復号やVPN処理などのインプレース暗号化操作中に、読み取り専用であるはずの実行ファイルやファイルを改ざんすることを可能にします。CAP_NET_ADMIN権限を持つローカルユーザーであればroot権限を奪取できるため、Kubernetesクラスター等のマルチテナント環境において極めて危険です。
[出典: SecurityWeek - 'DirtyClone' Linux Kernel Vulnerability Leads to Root Access]

### 2. 「Claude Code」を介した開発者端末の乗っ取り攻撃
Mozillaの0Dinセキュリティチームは、Anthropicの自律型開発エージェント「Claude Code」に対する「間接プロンプトインジェクション」をデモ公開しました。攻撃者は、一見無害なソフトウェアリポジトリを準備し、初期設定時にエラーを発生させます。AIエージェントがこのエラーを自律的に解決しようとリポジトリ内の指示に従った際、外部のDNS TXTレコードから取得された悪意あるペイロードが実行され、開発者のマシン上にリバースシェルが構築されます。
[出典: SecurityWeek - Researchers Demo New Claude Code Attack Using Harmless-Looking Repositories to Hijack Developer Machines]

### 3. オーミケンシ、サイバー攻撃により有価証券報告書の提出期限を延長
繊維メーカーのオーミケンシは、2026年3月に新興ランサムウェアグループ「The Gentlemen」による攻撃を受け、基幹システムが停止した影響で、2026年3月期の有価証券報告書の提出期限を9月30日まで延長申請しました。攻撃者は未修正のVPNゲートウェイから侵入し、データベースを暗号化しました。同社は現在もERPシステムの一部がオフラインであり、決算や請求、決算処理をすべて手作業で進めていますが、会計監査人による検証が期日までに完了しない事態となっています。
[出典: サイバーセキュリティインテリジェンス・レポート 2026年中期版]

## 深掘り

### 自律型AIツールへの「過剰な信頼」が生む新たな攻撃表面

2026年後半の脅威トレンドとして、開発者が利用するAIツールを「信頼のチェーン」の弱点とする手法が顕在化しています。

#### 診断機能の悪用とステルス性
「Claude Code」への攻撃が画期的なのは、**リポジトリ自体に悪意のあるコードやシェルスクリプトが含まれていない**点です。静的解析ツール（SAST）やウイルス対策ソフトをすり抜け、AIが「良かれと思って」実行するトラブルシューティングの手順に攻撃コードを紛れ込ませます。これは、AIの自律性が高まるほど、人間による監視の目が届かない「実行の空白」が生まれるリスクを示しています。

#### DNSを利用したペイロード配信
攻撃コードは外部のDNSインフラにBase64エンコードされた状態で隠されており、ディスク上や通信路上に平文のシグネチャを残しません。AIエージェントが標準的なシステムツールを使用してDNSルックアップを行う挙動は、ネットワーク監視でも「正当な名前解決」として扱われるため、検知が極めて困難です。

### [関連知識]

#### [用語解説]
* **DirtyClone**: Linuxカーネルのページキャッシュ汚染を伴う特権昇格脆弱性。2022年の「Dirty Pipe」と構造的に類似しています。
* **間接プロンプトインジェクション**: AIが外部データ（Webサイトやリポジトリ）を読み込む過程で、意図しない悪意ある指示を取り込んで実行してしまう攻撃手法。
* **バックアップ復元キー**: Signal等の暗号化アプリで履歴を復元するための鍵。これが盗まれると、アカウントを再作成しても攻撃者は永続的にメッセージを傍受可能です。

#### [今後の展望]
組織は、AIエージェントを「アイデンティティ」として認識し、人間と同様の最小権限の原則（Least Privilege）を適用する必要があります。また、サプライチェーンの透明性を高めるため、経済産業省とIPAが推進する**「SCS評価制度」**のような標準的なセキュリティ水準の検証枠組みが、取引条件の必須要件となっていくでしょう。

## 結論

2026年6月末の情勢は、**「共有されたインフラ」と「自律化したツール」**が最大の標的であることを示しました。KDDIやPeopleSoftの事例は、1箇所の綻びが数千万のユーザーや複数の大企業へ連鎖的に波及するリスクを露呈させました。また、AIエージェントへの攻撃は、これまでの「コードを守る」セキュリティから「AIの判断プロセスを守る」ガバナンスへの転換を求めています。侵害を完全に防げない現状において、パスキーのようなフィッシング耐性認証の導入や、段階的な復旧を実現するレジリエントなBCPの構築が不可欠です。
