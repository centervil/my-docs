# 2026年7月15日 サイバーセキュリティニュースまとめ

## 主要ポイント

* **Microsoft 7月月例パッチとゼロデイ脆弱性の悪用**: SharePoint Server (CVE-2026-56164) と ADFS (CVE-2026-56155) における特権昇格の脆弱性が、修正プログラム公開前から悪用されていたことが確認されました。

* **脆弱性公開プロセスの国際的制度化**: CISA、NSA、JPCERT/CC を含む国際的な連合が、脆弱性調整公開 (CVD) に関する共同ガイドラインを公開し、「Secure by Design」原則の定着を目指しています。

* **サイバー犯罪インフラへの地政学的圧力**: 米国政府は、ランサムウェア攻撃を支援したとして、VPN サービス「First VPN Service (1VPNS)」およびマルウェア秘匿ツールの販売者を初めて制裁対象に指定しました。

## 脆弱性とインフラ防衛

2026年7月15日から16日にかけて、OS や重要ミドルウェアの深刻な脆弱性対応が焦点となりました。Microsoft の更新プログラムでは、アイデンティティ管理の中核である ADFS や協力ツールの SharePoint が標的となり、MFA をバイパスして機密情報へアクセスされるリスクが浮き彫りになりました。また、18年以上前の脆弱性 (CVE-2008-4128) が依然としてロシア系アクター (Berserk Bear 等) に悪用されているとして、CISA が KEV カタログに追加し、レガシー設定の排除を求めています。

## 国内インシデントとサプライチェーンリスク

日本国内では、重要サービスや製造業を狙った攻撃が相次ぎました。日本最大のタクシー会社である日本交通が不正アクセスによりシステムを緊急遮断し、配車業務に影響が出たほか、村田製作所やニデックの海外子会社でも情報漏洩やランサムウェア被害が報告されています。また、学術系 EC サイト「アカデミコナビ」の侵害により、複数の大学に跨る17万件の個人情報が流出の危機に瀕しており、サードパーティ管理の難しさが再確認されています。

## AIと次世代の脅威

AI を活用したワークフローの脆弱性が新たな攻撃表面となっています。AI エージェント・プラットフォーム「Dify」において、他者のチャットログを閲覧可能な「DifyTap」脆弱性群が発見されました。また、開発環境を狙う攻撃も高度化しており、GitHub Actions の「actions/checkout」における「pwn request」攻撃への対策や、Rust のパッケージレジトリを狙う「CrateDepression」など、開発サプライチェーン全体が汚染の脅威にさらされています。

# 2026年7月15日 サイバーセキュリティニュース詳細

## ニュース紹介

### 1. Microsoft 2026年7月月例パッチ：アイデンティティ基盤へのゼロデイ攻撃
Microsoft は2026年7月のセキュリティ更新プログラムを公開し、多数の脆弱性を修正しました。特に SharePoint Server (CVE-2026-56164) と Active Directory フェデレーション サービス (ADFS) (CVE-2026-56155) の特権昇格脆弱性は、すでに野生下での悪用が確認されています。これらは組織のアイデンティティ管理の根幹を突くものであり、攻撃者は特権を奪取することで MFA を無効化し、機密データにフルアクセスする可能性があります。

[出典: JPCERT/CC - 2026年7月マイクロソフトセキュリティ更新プログラムに関する注意喚起](https://www.jpcert.or.jp/at/2026/at260020.html)
[出典: Global Cybersecurity Governance and the Evolving Threat Landscape (July 2026)]

### 2. 米政府、ランサムウェア支援の VPN サービス「1VPNS」を制裁
米国財務省外国資産管理局 (OFAC) は、ランサムウェアグループに匿名インフラを提供していた「First VPN Service (1VPNS)」とその管理者を制裁対象としました。このサービスは2014年から運営され、ログを保持しないことを謳い、医療機関や金融機関への攻撃に利用されてきました。また、マルウェアを検知から逃れさせる「クリプター」の販売者も同時に制裁されました。

[出典: The Hacker News - U.S. Sanctions First VPN Service and Malware Cryptor Seller Over Ransomware Support](https://thehackernews.com/2026/07/us-sanctions-first-vpn-service-and.html)

### 3. 日本交通における不正アクセスとシステム緊急遮断
2026年7月15日、日本交通株式会社は基幹システムへの不正アクセスを検知し、被害拡大防止のためにタクシー配車システムやハイヤー予約システムを含む IT インフラを緊急遮断したと発表しました。この遮断により、アプリを通じた配車や一部の決済業務に制限が生じています。重要インフラの一翼を担う交通システムの停止は、物理的な移動の自由にも直接的な影響を及ぼしました。

[出典: ScanNetSecurity - 日本交通が不正アクセスでシステムを緊急遮断、タクシー配車やハイヤー予約等に一部影響](https://scan.netsecurity.ne.jp/article/2026/07/15/55706.html)

## 深掘り

### 脆弱性調整公開 (CVD) の国際的制度化と「Secure by Design」

2026年7月の重要な転換点は、単なる技術的パッチ管理を超えた「防御の制度化」にあります。

#### 国際協力による CVD ガイドラインの策定
CISA、NSA、JPCERT/CC、および英・蘭のサイバーセキュリティセンターは、ソフトウェアメーカーがセキュリティ研究者と協力するための共同ガイドを公開しました。これは、脆弱性管理を企業の内部問題から「グローバルな共同責任」へと昇華させる試みです。研究者への「セーフハーバー (法的保護)」の提供や、透明性のあるトリアージプロセスの確立を求めています。

#### 認証プロトコルの強制的な堅牢化
Microsoft は 7月の更新で、Kerberos プロトコルにおけるレガシーな RC4 暗号を完全に無効化する「強制フェーズ」を開始しました (CVE-2026-20833)。これは、数十年にわたり攻撃の入り口となってきた古い技術債務を強制的に解消するものであり、組織は AES ベースの暗号化への即時移行を迫られています。

### [関連知識]

#### [用語解説]
*   **CVD (Coordinated Vulnerability Disclosure)**: 脆弱性情報を公開する前に、開発者と発見者が調整を行い、修正策が準備された段階で公表するプロセス。
*   **KEV (Known Exploited Vulnerabilities) カタログ**: CISA が管理する、実際に攻撃に悪用されていることが確認された脆弱性のリスト。連邦機関には一定期間内の修正が義務付けられます。
*   **Squidbleed (CVE-2026-47729)**: Squid プロキシのメモリからクリアテキストの HTTP リクエストが漏洩する脆弱性。2014年の Heartbleed に類する深刻な影響が懸念されています。

#### [今後の展望]
*   **エージェンティック AI セキュリティの台頭**: Dify の事例が示すように、AI ワークフロー自体を標的とする攻撃が増加するため、AI モデルの挙動を監視・制限するランタイム・コントロールの導入が標準化されるでしょう。
*   **金融・経済制裁によるサイバー犯罪抑止**: 1VPNS への制裁に見られるように、技術的防御が困難な匿名サービスに対し、経済的・法的手段でサプライチェーンを断つ戦略が加速します。

## 結論

2026年7月中旬の情勢は、従来の「境界防御」が限界を迎え、**「アイデンティティ (身元確認)」**と**「サプライチェーンの透明性」**が防御の主戦場となったことを示しています。SharePoint や ADFS へのゼロデイ攻撃は、信頼の根幹が常に狙われていることを物語っています。組織は、単なるパッチ適用に留まらず、CVD プログラムの構築やレガシープロトコルの廃止といった構造的な「Secure by Design」への転換、そして国内外のパートナーとの情報共有を深化させることが不可欠です。
