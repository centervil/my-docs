# 2026年5月9日 サイバーセキュリティニュースまとめ

## 主要ポイント

* **境界防御の限界と「レジリエンス」への転換:** CISAが発表した「CI Fortify」イニシアチブに見られるように、防御側は「侵入されないこと」から、攻撃を受けている最中でもサービスを継続する「隔離と回復」のモデルへと戦略をシフトさせています。

* **重要インフラとエッジデバイスの脆弱性悪用:** Palo Alto NetworksのPAN-OSやIvanti EPMMのゼロデイ脆弱性が国家背景アクターによって悪用されており、パッチが提供される前の「パッチギャップ」期間中のリスクが顕在化しています。

* **暗号の腐敗とサプライチェーンの汚染:** 台湾の高速鉄道が19年間の鍵更新なしでハッキングされた事例や、DAEMON Tools Liteの公式配布サイトがマルウェア配信に悪用された事例は、長期間の運用と信頼された配信経路に潜む盲点を示しています。

## 重要インフラの防衛と国家戦略

米CISAは、重要インフラ（CI）を保護するための新たな取り組み「CI Fortify」を開始しました。この活動は、国家背景アクターが既にインフラ内部に潜伏（pre-positioning）しているという前提に立ち、ITネットワークからOT（制御技術）を即座に切り離す「隔離（Isolation）」と、オフラインバックアップを用いた「回復（Recovery）」を柱としています。

## 境界デバイスおよびカーネルの高度な侵害

Palo Alto NetworksのPAN-OSに存在するバッファオーバーフロー脆弱性（CVE-2026-0300）が、 unauthenticated remote code execution (RCE) としてアクティブに悪用されています。また、Linuxカーネルでは「Copy Fail」の発展型とされる「Dirty Frag」脆弱性が特定され、決定論的なロジックバグにより多くのディストリビューションでroot権限の奪取が可能となっています。

## 国内動向とAI時代の新脅威

日本国内では、ランサムウェア被害を受けた企業の約70%がデータ復旧に失敗しているという実態が浮き彫りになりました。また、企業の独自LLMを操作して情報を盗み出す「LLM Jacking」のリスクや、自律型AIエージェントによる脆弱性調査（Agentic Discovery）への対抗策など、AIを巡る攻防が激化しています。

# 2026年5月9日 サイバーセキュリティニュース詳細

## ニュース紹介

### 1. Palo Alto Networks PAN-OS ゼロデイ脆弱性 (CVE-2026-0300) の悪用

PAN-OSのUser-ID認証ポータルにおいて、未認証の攻撃者がroot権限でコード実行を可能にするバッファオーバーフローが判明しました。CL-STA-1132と名付けられた国家背景の疑いがあるクラスターが、パッチ公開予定の約1ヶ月前からこの脆弱性を悪用して偵察やペイロードの投下を行っていました。CISAは、パッチのリリースを待たず連邦機関に対し緊急の対策（User-IDポータルの無効化等）を命じました。

[出典: Palo Alto PAN-OS Flaw Under Active Exploitation Enables Remote Code Execution](https://thehackernews.com/2026/05/palo-alto-pan-os-flaw-under-active.html)

### 2. 台湾高速鉄道：19年間放置された暗号鍵によるハッキング

23歳の大学生がソフトウェア無線 (SDR) を使用して台湾高速鉄道のシステムに侵入し、列車4台を緊急停車させました。この攻撃が成功した最大の要因は、列車無線に使用されていたTETRAシステムの暗号鍵が19年間一度も更新されておらず、古いTEA1暗号が実質的に形骸化していたことにあります。重要インフラにおけるライフサイクル管理と暗号の現代化の欠如が物理的な混乱を招いた象徴的事案です。

[出典: College student hacks Taiwan high-speed rail line with software defined radios, stopping four trains - Reddit](https://www.reddit.com/r/cybersecurity/comments/1t8p0jn/college_student_hacks_taiwan_highspeed_rail_line/)

### 3. DAEMON Tools Lite 公式インストーラーのサプライチェーン侵害

仮想ドライブ作成ソフト「DAEMON Tools Lite」の公式サイトで配布されていたバージョン12.5.1のインストーラーが、約1ヶ月間にわたりトロイの木馬化されていたことが判明しました。侵害されたインストーラーはデバイス情報の収集後にバックドアを設置する多段階のペイロードを含んでおり、ロシアやドイツ、中国などの政府・製造業を中心に約100組織が影響を受けました。現在は修正版の12.6.0.2445が公開されています。

[出典: RCE in PAN-OS and Ivanti EPMM Exploited: Check Exposure and Patch; DAEMON Tools Lite Supply Chain Compromised - SANS Institute](https://www.sans.org/newsletters/newsbites/xxviii-35)

## 深掘り

### 境界デバイスへの「パッチギャップ」と長期潜伏の脅威

2026年5月の状況は、組織のセキュリティの要であるファイアウォールやVPN機器自体が、最も脆弱な急所となっていることを示しています。

#### パッチの遅れと実悪用の先行
PAN-OSの事例では、悪用が確認された5月6日時点で修正プログラムは未リリースであり、5月13日までパッチが提供されない「パッチギャップ」が生じていました。この間、防御側は機能を停止するか制限をかけるという、可用性を犠牲にした対応を余儀なくされました。

#### 高度な隠蔽と永続化
「PamDOORa」バックドアやCisco機器を狙う「FIRESTARTER」のように、パッチ適用やOS再起動後も、PAMモジュールやファームウェアレベルで潜伏し続ける技術が一般化しています。これにより、一度侵害された機器の完全性はパッチだけでは回復できず、初期化（再イメージング）が必要となる事態が発生しています。

### 関連知識

#### 用語解説

* **Dirty Frag:** Linuxカーネルのページキャッシュ書き込みロジックを突く脆弱性。タイミングに依存しない確定的な攻撃が可能で、root権限取得の成功率が非常に高い。
* **CI Fortify:** 米CISAが発表した重要インフラ向けイニシアチブ。「攻撃を受けている最中でもサービスを維持すること」を目的とし、隔離と回復を重視する。
* **LLM Jacking:** 組織内のAIアシスタントを操作し、本来アクセスできない機密情報を抽出したり、不正なアクションを実行させたりする攻撃手法。

#### 今後の展望

今後は「侵害されることを前提（Assume Breach）」とした設計への移行が加速します。重要インフラや医療機関においては、予算不足を理由にした対策の遅れは許容されなくなり、デジタル・トランスフォーメーション (DX) の初期段階からセキュリティを組み込むことが法規制等により義務化される方向です。また、AIを用いた脆弱性の自律的発見に対抗するため、防御側も「エージェント型AI」による自律的な検証・修復システム（Agentic Validation）の導入が不可欠となります。

## 結論

最新のソースは、**「境界防御の神話が崩壊し、インフラの深部にまで脅威が到達している」**現状を示しています。PAN-OSやIvantiのゼロデイ悪用、および長期間放置された暗号鍵による台湾の鉄道事故は、パッチ適用という受動的な対応の限界を露呈させました。組織はCISAの「CI Fortify」が提唱するように、攻撃を受けた状態でも中核サービスを維持できる「マシン速度のレジリエンス」の構築、およびサプライチェーンやAI資産に対する厳格なガバナンス体制への刷新を、経営最優先の課題として取り組む必要があります。
