# 2026年4月4日 サイバーセキュリティニュースまとめ

## 主要ポイント

*   **信頼された供給網の崩壊とAxiosへの大規模攻撃**: 週1億回以上のダウンロードを誇るnpmパッケージ「Axios」が北朝鮮の「Sapphire Sleet」により侵害され、インストール直後にシステムを掌握するクロスプラットフォームRATが配布されました。
*   **AIエージェント「Claude Code」のソース流出と脆弱性の露呈**: Anthropic社のAI開発ツール「Claude Code」の全ソースコード約51万行が公開レジストリに流出し、さらに「禁止ルール」を回避可能な深刻な設計上の脆弱性も発見されました。
*   **重要インフラを狙うCitrix NetScalerのゼロデイ悪用**: 管理者セッションIDの窃取を可能にする脆弱性（CVE-2026-3055）が実環境で悪用されており、CISAは連邦政府機関に対して数日以内の緊急パッチ適用を命じています。

## ソフトウェアサプライチェーンの脅威
2026年3月末の短期間に、Trivy、LiteLLM、Axiosといった主要なオープンソースプロジェクトが相次いで侵害される未曾有の事態が発生しています。これらの攻撃は、**開発・CI/CD環境で高い権限を持つツールを標的**にしており、一度の侵害で数百万のダウンストリーム環境に連鎖的な被害を及ぼす「連鎖的な信頼の悪用」が特徴です。

## AI技術の脆弱性とガバナンスの変容
自律型AIエージェントがシステムレベルの実行権限を持つようになったことで、新たな攻撃面が顕在化しています。Claude Codeで見つかった「50件以上のサブコマンドで制限を回避できる」脆弱性は、**AIの利便性（パフォーマンス最適化）とセキュリティ（権限管理）のトレードオフ**が、攻撃者にとっての絶好の隙間となっていることを示しています。

## 国内外の組織体制とレジリエンス
日本国内では、2026年4月1日の組織改編に伴い、JPCERT/CCが脆弱性公表から悪用までの短縮化（Nデー脅威）に対応するため、一本化された「サイバーセキュリティコーディネーショングループ」を発足させました。また、アサヒグループや中小企業におけるランサムウェア被害の教訓から、**オフラインバックアップの確保とITシステム停止を前提としたBCP**の重要性が再認識されています。

# 2026年4月4日 サイバーセキュリティニュース詳細

## ニュース紹介

### 1. Axios npmパッケージ侵害：北朝鮮グループによる高度なサプライチェーン攻撃
2026年3月31日、JavaScriptの主要ライブラリ「Axios」のメンテナアカウントが北朝鮮背景の「Sapphire Sleet」に奪取され、悪意ある依存関係を注入されたバージョン（1.14.1および0.30.4）が公開されました。この攻撃は単純な認証情報漏洩ではなく、**数週間にわたる巧妙なソーシャルエンジニアリング**（偽のSlackやTeams会議による誘導）を通じて行われ、MFAをも回避するレガシーな公開トークンが利用されました。感染すると、Windows、macOS、Linuxそれぞれに最適化されたRATが自動配備され、数秒以内にクラウドアクセスキーやSSHキーなどの機密情報が窃取されます。
[出典: Microsoft Security Blog - Mitigating the Axios npm supply chain compromise / SANS Institute - Axios NPM Supply Chain Compromise]

### 2. Anthropic、設定ミスにより「Claude Code」の全ソースコードを流出
AI開発大手Anthropicは、Bundlerとして採用した「Bun」の既知のバグと人為的ミスが重なり、主力製品「Claude Code」の全TypeScriptソースコード約512,000行を誤って公開npmレジストリにアップロードしました。この流出により、AIエージェントの**「3層メモリ構造」や未発表機能（KAIROS、ULTRAPLAN等）の詳細**が露呈しました。さらに、このソースコードを悪用して、「Unlocked（制限解除版）」を装ったマルウェア（VidarやGhostSocks）を配布する偽のGitHubリポジトリも確認されており、二次被害が拡大しています。
[出典: NodeSource - Anthropic Accidentally Leaked Claude Code's Entire Source / Zscaler - Anthropic Claude Code Leak]

### 3. Citrix NetScalerの重大な脆弱性（CVE-2026-3055）への警告
Citrix NetScaler ADCおよびGatewayにおいて、認証なしでメモリ内の機密情報を読み取ることができる深刻な脆弱性（CVE-2026-3055、CVSS 9.3）が報告されました。この脆弱性は、2023年に猛威を振るった「CitrixBleed」と技術的に類似しており、**SAML Identity Provider (IdP) として構成されている場合に、管理者セッションIDや資格情報の漏洩**を招く恐れがあります。すでに実環境での悪用（ハニーポットへの攻撃試行）が観測されており、CISAは連邦政府機関に対し、2026年4月2日までの即時パッチ適用を命じました。
[出典: Rapid7 - CVE-2026-3055: Citrix NetScaler ADC / The Record - CISA tells federal agencies to patch Citrix NetScaler bug]

## 深掘り

### 「信頼の武器化」とアイデンティティ中心の防御
AxiosやClaude Codeの事例は、組織が最も信頼しているツールやプロセスそのものが最大の攻撃ベクトルになるというパラドックスを証明しています。

#### 供給網における「混合公開ワークフロー」の死角
Axiosの事例では、組織がMFAやOIDCといった強固な管理を導入していても、**レガシーな認証パス（長寿命のnpmトークン）が並存していること**が致命的な隙となりました。攻撃者は、最も強固な壁を壊すのではなく、残された最も脆弱な経路を選択します。これは、単に新しい技術を導入するだけでなく、古い認証方式を完全に廃止し、エンドツーエンドで管理を徹底することの難しさを示しています。

#### エージェンティックAIにおける「サイレントバイパス」
Claude Codeの脆弱性分析では、AIが複雑な処理（50件以上のサブコマンド）を行う際に、**「安全性の証明が困難なためプロンプトで確認する（ask）」というフォールバック処理**が、事実上の「禁止ルール（deny）の無効化」として機能していたことが判明しました。攻撃者がAIを誘導し、一見無害な多くのステップの後に悪意あるコマンドを忍び込ませることで、ユーザーは警告に気づかず「許可」を与えてしまうリスクがあります。これは、AIを「使うツール」から「自律的に動くエージェント」へと進化させる際に、人間側がどのような判断基準でAIを監視すべきかという新たな課題を突きつけています。

### [関連知識]

#### 用語解説
*   **WAVESHAPER.V2**: 北朝鮮の攻撃グループが使用する、システム情報の収集や任意のコマンド実行を可能にする高度なバックドア。
*   **Bun Bug #28001**: Bunランタイムにおいて、プロダクションモードでもソースマップを強制的に生成してしまう既知のバグ。今回のClaude Code流出の直接的な原因の一つとなりました。
*   **Nデー脅威**: 脆弱性が公表された直後（あるいはパッチ公開直後）に、攻撃者が即座にそれを武器化して攻撃を開始する事態。

#### 今後の展望
今後は、AIエージェントを「見えない特権ユーザー」として扱う**「Agentic Zero Trust」モデル**の構築が必須となります。開発環境では `--ignore-scripts` の強制や、短期間のパッケージ隔離（Quarantine）ポリシーの導入といった、利便性と速度を多少犠牲にしても「信頼を検証する」ためのアーキテクチャレベルでのレジリエンスが求められるようになるでしょう。

## 結論
2026年3月から4月にかけての情勢は、サイバー攻撃が「技術の隙」を突くものから、**「人間の信頼とAIの自律性」を組織的に悪用するもの**へと完全に移行したことを示しています。Axiosの侵害やClaude Codeの流出、そしてNetScalerの悪用は、どれも「信頼されていたはずの基盤」から発生しています。組織は侵害を前提（Assume Breach）とし、リアルタイムのアイデンティティ監視と、AIを含むすべての実行主体に対する厳格なガバナンスを統合した、新たなセキュリティ基準を確立することが最優先課題です。
