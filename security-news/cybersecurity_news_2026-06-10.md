# 2026年6月10日 サイバーセキュリティニュースまとめ

## 主要ポイント

* **九州大学、ランサムウェア攻撃により患者の手術動画等が流出の恐れ**: 研究室のPCが感染し、大学病院の患者43人分の氏名や手術動画データが外部へ流出した可能性があります。

* **元従業員による大規模な個人情報持ち出しが発覚**: 株式会社Kaizen Tech Agentにて、元従業員が在職時のアクセス権限を悪用し、約9,700件の個人情報を不正取得・持ち出した疑いが報じられました。

* **Check Point VPNおよびGoogle Chromeのゼロデイ脆弱性を突く攻撃**: VPNの認証バイパス（CVE-2026-50751）がランサムウェア攻撃に悪用されているほか、Chromeでも2026年で5件目となるゼロデイが修正されました。

## 国内インシデントとアイデンティティ管理

国内では、組織の「内部」と「境界」の両面で深刻な事案が発生しています。九州大学の事例では、学術研究用ネットワークと臨床用ネットワークの境界管理が課題として浮き彫りになりました。また、Kaizen Tech Agentの事案は、退職者のアカウント権限を即座に失効させるアイデンティティ・ライフサイクル管理の不備を突かれた形です。さらに、大阪・関西万博のチケットサイトへの不正ログインによる高額転売容疑での逮捕者が出るなど、リスト型攻撃（クレデンシャル・スタッフィング）による公的サービスへの被害も現実化しています。

## 脆弱性トレンドと重要インフラの危機

境界防御製品を標的としたゼロデイ攻撃が激化しています。Check Point製VPNの脆弱性は、古いプロトコル（IKEv1）の検証不備を突くもので、Qilinランサムウェアグループによる悪用が確認されています。また、ITレイヤーのみならず、物理インフラを支えるデータセンターのHVAC（空調）やUPS（無停電電源装置）の脆弱性も報告されました。これらが悪用されると、リモートからサーバー室の温度を操作し、ハードウェアの物理的破壊やサービス停止を引き起こす「サイバー・フィジカル攻撃」が可能になります。

## 2026年の脅威展望と産官学の対応

IPAが公開した「10大脅威 2026」では、組織編の3位に「AIの利用をめぐるサイバーリスク」が初めてランクインしました。AIは攻撃の高度化（フィッシング文面の精巧化など）に寄与する一方、OpenSSLの脆弱性発見にAIが活用されるなど、防御側のツールとしての二面性を持っています。日本政府はサプライチェーン全体の耐性を高めるため、委託先のセキュリティ水準を評価する「SCS評価制度」の運用準備を加速させており、中小企業を含むエコシステム全体の底上げが図られています。

# 2026年6月10日 サイバーセキュリティニュース詳細

## ニュース紹介

### 1. 九州大学病院の患者情報および手術動画の流出事案
2026年5月25日、九州大学の研究室が管理するPCがランサムウェアに感染しました。教員が使用中に画面がロックされ、金銭を要求する脅迫文が表示されました。当該端末には大学病院の患者43人分の氏名や手術動画データが保存されており、これらが外部へ流出した可能性があります。大学側は金銭の要求には応じない方針を示しており、診療システム自体はネットワークが切り離されていたため影響はないとしています。
[出典: KBC九州朝日放送 - 九州大学のパソコンがサイバー攻撃 大学病院の患者データ流出の可能性](https://kbc.co.jp/news/article.php?id=17517628&ymd=2026-06-10)

### 2. Kaizen Tech Agentにおける元従業員による内部不正
人材紹介会社の株式会社Kaizen Tech Agentは、元従業員が業務上のアクセス権限を悪用し、2025年10月に約9,721件の個人情報を不正に取得・持ち出した疑いがあると発表しました。漏洩した可能性のある情報には、氏名、住所、銀行口座情報のほか、健康状態等を含む「面談時の記録」という極めて機微な情報が含まれています。2026年3月の社内調査で発覚し、同社は警察への相談および個人情報保護委員会への報告を実施しました。
[出典: 株式会社Kaizen Tech Agent - 個人情報の漏洩の疑いに関するご報告とお詫び](https://kaizentech.co.jp/5388/)

### 3. Check Point VPNのゼロデイ脆弱性（CVE-2026-50751）の悪用
Check Pointは、自社のVPNおよびファイアウォール製品に存在する深刻な認証バイパスの脆弱性が、ゼロデイ攻撃として悪用されていると警告しました。この脆弱性は、古いIKEv1プロトコルを使用している環境で、有効なパスワードなしにVPN接続を確立できてしまう論理的な欠陥です。すでにQilinランサムウェアの提携組織による悪用が確認されており、米CISAも「悪用が確認された脆弱性カタログ（KEV）」に追加して迅速なパッチ適用を求めています。
[出典: SecurityWeek - Check Point VPN Zero-Day Exploited in Qilin Ransomware Attacks](https://www.securityweek.com/check-point-vpn-zero-day-exploited-in-qilin-ransomware-attacks/)

## 深掘り

### 境界防御の限界とアイデンティティ・ガバナンス

2026年6月のインシデントは、従来の「境界を固める」防御モデルが複数の方向から崩しにかかられていることを示しています。

#### IDライフサイクル管理の盲点
Kaizen Tech Agentの事例は、従業員の退職と連動した「自動化された権限削除」の重要性を強調しています。手動の運用では、削除漏れや遅延が発生し、悪意を持った（あるいは買収された）元関係者によるデータ窃取を許してしまいます。

#### ネットワーク・セグメンテーションの再定義
九州大学の事案では、柔軟性が求められる「学術研究網」と、機密性が求められる「臨床・医療網」が物理的・論理的に不十分にしか隔離されていなかった可能性が指摘されています。ランサムウェアが研究用端末から患者データにリーチできた事実は、ゼロトラストに基づく厳格なセグメンテーションと、多要素認証（MFA）の全面的な適用が不可欠であることを示唆しています。

### 関連知識

#### 用語解説
*   **IKEv1（Internet Key Exchange version 1）**: VPN接続時の鍵交換プロトコル。現在は古く（非推奨）、設計上の脆弱性が指摘されているが、後継のIKEv2へ移行していないレガシー環境が今回のCheck Pointの事案で狙われました。
*   **SCS評価制度（Supply Chain Security Evaluation System）**: 経済産業省とIPAが主導する、サプライチェーン上の企業のセキュリティ対策を「星」で格付け・認証する制度。2026年度から本格的な詳細化と運用開始が予定されています。
*   **ヒープ・ユーザー・アフター・フリー（Heap User-after-free）**: メモリ管理の脆弱性の一種。解放した後のメモリ領域を再度使用することで、プログラムのクラッシュや任意のコード実行を引き起こします。最新のOpenSSLやChromeの脆弱性でも関連する不備が報告されています。

#### 今後の展望
AIを活用した脆弱性発見が加速する中、修正パッチが公開されてから攻撃が始まるまでの「悪用までの時間（Time to Exploit）」が劇的に短縮されています。組織は「パッチ管理」という後追い対策だけでなく、不要なプロトコル（IKEv1等）の廃止、物理インフラ（HVAC/UPS）のITネットワークからの論理的隔離、そして「SCS評価制度」などを通じたサプライチェーン全体の透明性確保を優先すべきフェーズに突入しています。

## 結論

2026年6月の情勢は、ランサムウェアグループによるゼロデイ脆弱性の即時悪用と、内部特権の悪用という「内外からの挟み撃ち」が常態化していることを示しています。特に医療やインフラといった社会の基盤を支える組織において、データの保存場所を「誰が」「いつまで」触れるのかをシステム的に制御し、物理的な設備（空調・電源）までを含めた包括的なレジリエンス設計を再構築することが、2026年の最優先課題です。
