# 2026年4月3日 サイバーセキュリティニュースまとめ

## 主要ポイント

* **オープンソース供給網の信頼モデル崩壊とAxiosへの大規模攻撃:** 週1億回以上のダウンロードを誇るnpmパッケージ「Axios」が北朝鮮背景の「Sapphire Sleet」により侵害され、インストールからわずか15秒でシステムを掌握するクロスプラットフォームRATが配布されました。
* **AIエージェント「Claude Code」のソースコード流出と論理脆弱性:** Anthropicが設定ミスにより自社ツールのソースコード約51万行を漏洩させたほか、50個以上のサブコマンドを実行するとセキュリティチェックがサイレントにバイパスされる深刻なロジック上の欠陥が発覚しました。
* **重要インフラのアイデンティティ境界を突くゼロデイ攻撃:** Citrix NetScalerにおけるメモリ領域のデータ読み取りを許す深刻な脆弱性（CVE-2026-3055）が確認され、CISAは連邦機関に対し緊急のパッチ適用を命じました。

## ソフトウェアサプライチェーンの武器化

2026年3月末、JavaScriptエコシステムの基盤であるAxiosが、攻撃者の配信拠点へと変貌しました。この攻撃では、Axios自体のコードを一行も書き換えずに「ファントム依存関係（実体のない依存関係）」を追加する手法が採られ、開発者が`npm install`を実行した瞬間にインストール時スクリプト（postinstall）を通じてマルウェアが配備されます。これにより、GitHub ActionsなどのCI/CDパイプライン上でクラウド認証情報や秘密鍵が即座に窃取されるリスクが現実のものとなりました。

## 自律型AIエージェントの露出と管理不能なリスク

Anthropicの「Claude Code」の漏洩は、AI企業の「安全性第一」という掲示と、実際の開発管理プロセスとの乖離を浮き彫りにしました。流出したコードからは、未公開の自律型プロジェクト「KAIROS」の詳細が判明したほか、AIエージェントが高い権限を持つ一方で、計算コスト（トークン消費）削減のためにセキュリティ検証をスキップする「トレードオフの罠」が存在することが明らかになりました。これは、AIが「正規の操作」を模倣しながら悪意ある行動を取る「コンフューズド・デピュティ（混乱した代理人）」問題の深刻化を象徴しています。

## エンタープライズインフラと日本国内の動向

企業のアイデンティティとリモートアクセスの要であるCitrix NetScalerで発見された脆弱性（CVE-2026-3055）は、過去の「CitrixBleed」と類似した手法で管理者のセッション情報を窃取できるため、極めて高い優先度での対応が求められています。日本国内では、JPCERT/CCが脆弱性公表から悪用までの「猶予時間の短縮」に対応するため、早期警戒とインシデント対応の組織統合を実施しました。また、国内中小企業のIT資産全てに何らかの脆弱性が検出されるという衝撃的な実態調査も報告されています。

# 2026年4月3日 サイバーセキュリティニュース詳細

## ニュース紹介

### 1. Axios npmパッケージ侵害、北朝鮮グループによる大規模サプライチェーン攻撃
2026年3月31日、JavaScriptの主要ライブラリ「Axios」のメンテナアカウントが北朝鮮背景の脅威アクター「Sapphire Sleet（別名：BlueNoroff、UNC1069）」により奪取されました。攻撃者は、悪意ある依存関係「plain-crypto-js@4.2.1」を注入したバージョンを公開し、npmのpostinstallフックを利用して、Windows、macOS、Linuxそれぞれに対応したRAT（遠隔操作ツール）を配備しました。この攻撃は非常に周到で、実行後に自身の痕跡を削除しpackage.jsonを正常なものに差し替えるといった高度な隠蔽工作が行われていました。
[出典: Trend Micro - Axios NPM Package Compromised] [出典: Microsoft Security Blog - Mitigating the Axios npm supply chain compromise]

### 2. Anthropic、設定ミスにより「Claude Code」の全ソースコード流出と論理バグ発覚
AI開発大手のAnthropicは、開発ツール「Bun」のソースマップ生成に関する設定ミスにより、主力製品「Claude Code」の全TypeScriptソースコード約513,000行を誤って公開npmレジストリにアップロードしました。この流出により、自律型エージェントの核心技術である「Tengu」アーキテクチャや未公開機能が露呈しました。さらに、AIセキュリティ企業Adversaは、50個以上のサブコマンドを連結して実行させると、計算負荷軽減のためにセキュリティ・ブロックルールが無視されるという、設計上の致命的な脆弱性を指摘しました。
[出典: XenoSpectrum - Anthropic「Claude Code」ソースコード全容が流出] [出典: Adversa AI - Critical Claude Code vulnerability]

### 3. Citrix NetScalerの重大なメモリ漏洩脆弱性（CVE-2026-3055）
Citrix NetScaler ADCおよびGatewayにおいて、認証なしでメモリから機密情報を窃取可能な境界外読み取りの脆弱性が発見されました。この脆弱性は、デバイスがSAML Identity Provider (IDP) として構成されている場合に、細工されたリクエストを通じて管理者のセッションIDなどを露出させます。既に野生下での悪用が確認されており、CISAはすべての連邦機関に対し、2026年4月2日までの即時修正を命じました。
[出典: Penligent - CVE-2026-3055 and the NetScaler Identity Edge] [出典: JPCERT/CC - 注意喚起（CVE-2026-3055）]

## 深掘り

### 信頼の自動化が招く「15秒の侵害」：サプライチェーン攻撃の新段階

2026年3月末のAxiosへの攻撃は、現代のソフトウェア開発が抱える「信頼の連鎖」の脆弱性を極限まで露呈させました。

#### ファントム依存関係：見えない侵入
今回の攻撃の核心は、Axios自体のソースコードを一行も書き換えることなく、npmの依存関係解決機能を悪用した点にあります。攻撃者は「plain-crypto-js」という無害を装った偽パッケージを仲介させ、開発者が`npm install`を実行した際の自動処理（ポストインストールスクリプト）を武器化しました。インストールから資格情報の奪取完了までわずか15秒というスピードは、人間による監視が介在する余地を完全に奪い去っています。

#### AIエージェントにおけるコストと安全性の対立
Claude Codeの脆弱性は、AI業界が直面する構造的な課題を浮き彫りにしました。セキュリティ検証は計算コスト（トークン）を消費するため、パフォーマンス向上のために検証ステップを簡略化した結果、攻撃者が意図的に「検証の限界（50コマンド）」を超える命令を送り込むことで、設定された禁止ルールを完全に無効化できることが証明されました。

### [関連知識]

#### [用語解説]
*   **WAVESHAPER.V2:** 北朝鮮の脅威アクターがAxios攻撃に使用した高度なRAT。JSONベースの通信を行い、OSを自動判別してメモリ上でコードを実行します。
*   **KAIROS:** 流出したClaude Codeのソースから発見された次世代プロジェクト。バックグラウンドで常駐し、自律的にエラー修正などを提案する「プロアクティブなエージェント」構想です。
*   **SBOM (Software Bill of Materials):** ソフトウェア部品表。Axiosのような「依存関係の裏口」による侵害を検知するために不可欠な管理ツールとして、その重要性が再認識されています。

#### [今後の展望]
2026年度、組織は「パッチ管理」という受動的な対応から、AIを活用した異常検知とインシデント対応の自動化をBCP（事業継続計画）の中核に据える必要があります。また、AIエージェントにシステム権限を与える際は、エージェント自身のチェック機能に依存せず、OSレベルでのサンドボックス化や「人間による明示的な承認」を技術的に強制する「エージェント・ゼロトラスト」の構築が企業の生存戦略となるでしょう。

## 結論

最新の情勢は、**「組織が最も信頼しているツールやプロセス（常用ライブラリ、AIエージェント、アイデンティティ管理）」こそが、最大の攻撃ベクトルになる**というパラドックスを証明しました。Axiosの侵害、Claude Codeの流出、NetScalerの脆弱性は、それぞれ開発者のPC、AIのロジック、インフラの境界という「信頼の根幹」を同時に突いています。組織は「侵害を前提（Assume Breach）」としたレジリエンス構築を最優先し、AI駆動の自律的な防御と、厳格なアイデンティティ管理を統合した新たなセキュリティ基準の確立を急がなければなりません。
