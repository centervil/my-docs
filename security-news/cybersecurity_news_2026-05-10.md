# 2026年5月10日 サイバーセキュリティニュースまとめ

## 主要ポイント

* **AIによる脆弱性発見の自動化と加速**: AnthropicのAIモデル「Mythos」の登場により、ゼロデイ脆弱性の発見から武器化までの時間が数週間から1時間未満へと劇的に短縮される「脆弱性の津波」が到来しています。

* **Linuxカーネルにおける深刻な特権昇格脆弱性の露呈**: 「Dirty Frag」や「Copy Fail」といった、クラウドやコンテナ環境に甚大な影響を及ぼすローカル特権昇格（LPE）脆弱性が相次いで報告され、実環境での悪用も確認されています。

* **国内企業のインシデント報告とゼロトラストへの転換**: アサヒグループHDがランサムウェア被害の最終報告で「VPNの全面廃止」とゼロトラスト移行を表明し、村田製作所も大規模な情報流出の可能性を公表するなど、国内大手の被害実態と対策強化が鮮明になっています。

## AIと次世代の脅威

Anthropicが発表した「Claude Mythos」は、自律的にゼロデイ脆弱性を発見・連鎖させる能力を持ち、従来のパッチ管理サイクルを無効化するほどの衝撃を与えています。これを受け、オーストラリアの規制当局ASICは金融セクターに対し、AIによる攻撃の迅速化に備えるよう緊急の警告を発しました。また、AIエージェントの普及に伴い、ブラウザ拡張機能の脆弱性を突く「ClaudeBleed」のような新たな攻撃手法も報告されています。

## 基盤ソフトウェアの脆弱性

Linuxカーネルの「Copy Fail」(CVE-2026-31431)は、暗号化サブシステムの論理的な欠陥を突き、わずか732バイトのスクリプトでroot権限を奪取可能です。続いて登場した「Dirty Frag」(CVE-2026-43284等)も、メモリ断片化管理の不整合を悪用して信頼性の高い特権昇格を実現します。これらの脆弱性はコンテナ環境での「脱獄」やマルチテナント環境の侵害を容易にするため、クラウド基盤にとって極めて深刻な脅威となっています。

## 国内企業の動向とアイデンティティ管理

国内では、アサヒグループHDが5ヶ月にわたる物流混乱を経て、VPN廃止という抜本的なセキュリティ改革を打ち出しました。村田製作所は従業員や取引先など約8.8万件の情報流出の可能性を公表しています。一方で、Keeper Securityの調査によれば、日本のITリーダーの85%がAI拡大によるアイデンティティ管理の増加に限界を感じており、リアルタイム検知への自信が世界平均と比較して著しく低いことが明らかになりました。

## ニュース紹介

### 1. AnthropicのAIモデル「Mythos」がセキュリティ環境を激変

Anthropicが公開したフロンティアAIモデル「Claude Mythos」は、高度なコーディング能力と推論力を備え、自律的にゼロデイ脆弱性を発見・悪用することができます。レッドチームの演習では、OpenBSDに27年間存在した脆弱性を2万ドル以下のコストで発見しました。このモデルの登場により、脆弱性の発見から攻撃コードの作成までの時間は数時間、将来的には1時間以内へと短縮されると予測されており、従来の人間による分析やパッチ適用では追いつけない「脆弱性の津波」を引き起こしています。

### 2. Linuxカーネルの脆弱性「Dirty Frag」および「Copy Fail」の脅威

Linuxカーネルにおいて、ページキャッシュを汚染することでroot権限を取得できる深刻な脆弱性が立て続けに発見されました。「Copy Fail」(CVE-2026-31431)は2017年から存在するメモリ処理の不整合を悪用するもので、実環境での悪用が確認されCISAのKEVカタログに追加されています。一方、「Dirty Frag」(CVE-2026-43284等)はネットワークモジュールのメモリ断片化処理を悪用し、従来のレースコンディションに依存する攻撃よりも確実に特権昇格を成功させます。これらはホストとコンテナでページキャッシュを共有する仕組みを悪用するため、クラウド環境全体の侵害につながるリスクがあります。

[出典: Microsoft Security Blog - CVE-2026-31431: Copy Fail vulnerability enables Linux root privilege escalation across cloud environments](https://www.microsoft.com/en-us/security/blog/2026/05/01/cve-2026-31431-copy-fail-vulnerability-enables-linux-root-privilege-escalation/)
[出典: Microsoft Security Blog - Active attack: Dirty Frag Linux vulnerability expands post-compromise risk](https://www.microsoft.com/en-us/security/blog/2026/05/08/active-attack-dirty-frag-linux-vulnerability-expands-post-compromise-risk/)

### 3. アサヒグループHDと村田製作所がサイバー攻撃の最終報告と現状を公表

アサヒグループHDは、2025年9月のランサムウェア攻撃に関する最終報告を公開しました。攻撃者はVPN経由で侵入後、管理者権限を奪取してデータセンター内のシステムを暗号化しました。同社は再発防止策として、侵入経路となったVPN装置を全面的に廃止し、ゼロトラストモデルへ完全移行することを決断しました。また、村田製作所は第三者による不正アクセスにより、従業員の住所や銀行口座、顧客との契約書類を含む約8.8万件のデータが流出した可能性があると発表しました。

## 深掘り

### 脆弱性の津波と防御のパラダイムシフト

AIによる脆弱性発見能力の爆発的な向上は、従来の「発見・分析・評価・パッチ」という防御サイクルを根本から破壊しつつあります。

#### 脆弱性管理の限界
NISTがすべての脆弱性を分析することを断念し、緊急性の高い案件のみを優先する方針に転換したことは、AI時代の「検知量の増加」に人間が対応できなくなっている現実を象徴しています。CVEリストに基づく従来の管理は、Mythosが発見する未公開（ゼロデイ）脆弱性に対しては無力です。

#### 実行時保護（Runtime Security）への移行
AIの速度に対抗するためには、静的なスキャンから「実行時に実際に悪用可能か」を判断するリスクベースの運用への転換が不可欠です。具体的には、稼働中のコードのみを対象とした脆弱性分析や、ビジネスへのインパクトを考慮した自動的な優先順位付けが求められます。

### 関連知識

#### 用語解説
* **フロンティアAI (Frontier AI)**: Mythosのように、既存のAIを凌駕する高度な推論やコーディング能力を持つ最先端のAIモデルを指します。
* **LPE (Local Privilege Escalation)**: ローカル特権昇格。一般ユーザー権限でログインした攻撃者が、OSの脆弱性を利用して管理者(root)権限を取得することを指します。
* **ゼロトラスト (Zero Trust)**: 「何も信頼しない」ことを前提に、すべてのアクセスに対して認証・認可を厳格に行うセキュリティの考え方。アサヒグループHDがVPN廃止後に採用したモデルです。

#### 今後の展望
今後、AI discoveryツールがサイバー犯罪者に普及することで、攻撃の「自動化・高速化」がさらに進むと予想されます。防御側も「Anya」のようなエージェント型AIを導入し、検知から修復までの自動化(MTTRの短縮)を図ることが生き残りの条件となるでしょう。

## 結論

2026年5月現在のセキュリティ環境は、AIによる「攻撃の自動化」と「基盤ソフトウェアの脆弱性」の二重の脅威に直面しています。特にAIモデル「Mythos」の登場は、脆弱性管理のあり方を「全件対処」から「リスクベースの優先順位付けと自動応答」へと強制的にシフトさせました。アサヒグループHDのような国内大手の事例は、もはや従来の境界防御では不十分であることを示しており、ゼロトラストへの移行とAIを活用した高度な監視体制の構築が、企業にとって急務となっています。
