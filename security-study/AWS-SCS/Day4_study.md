## 📖 解説パート

### 組織設計とアカウント分離戦略

AWS Organizationsを活用することで、複数のAWSアカウントを一元管理し、ポリシーを適用できます。重要なのは、目的に応じて「組織単位（OU）」を適切に設計することです。一般的に、以下の階層構造が推奨されます。

1. **本番環境OU**: サービスの本番稼働用アカウントを配置。厳格なアクセス制御を適用します。
2. **開発・検証環境OU**: 実験的な試行を許可しつつ、コストやセキュリティの範囲を制限します。
3. **セキュリティOU**: セキュリティ関連のログ集約や、監査用のアカウントを隔離して配置します。
4. **インフラOU**: ネットワークや共有サービスのアカウントを管理します。

アカウントを分離することで、万が一侵害が発生した場合の被害範囲（ブラスト半径）を最小化できます。また、各OUに対して異なるSCPを適用することで、環境の性質に応じた柔軟なセキュリティ管理が可能になります。

### SCPによるガードレールの設定

SCPは、OUやアカウントに対して「何ができるか」を制限するポリシーです。IAMポリシーと異なり、SCP自体は権限を許可せず、あくまで「許可の最大範囲」を制限するガードレールとして機能します。

例えば、「S3のパブリック公開を禁止する」といったSCPを適用すれば、たとえ管理者であってもそのポリシーに反する操作は実行できません。これにより、組織全体で統一されたセキュリティ基準を強制できます。

#### 実践：東京リージョン以外を制限するSCP

特定のリージョン以外でのリソース作成を禁止するSCPの例です。`Condition`句で`aws:RequestedRegion`を指定することで、指定外のリージョンへのアクセスを拒否します。

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DenyAllOutsideTokyo",
      "Effect": "Deny",
      "NotAction": [
        "iam:*",
        "support:*",
        "route53:*"
      ],
      "Resource": "*",
      "Condition": {
        "StringNotEquals": {
          "aws:RequestedRegion": "ap-northeast-1"
        }
      }
    }
  ]
}
```

#### 重要ポイント

* **権限の境界**: SCPはIAMポリシーの「拒否」を上書きできないため、常に制限として機能する。
* **階層の継承**: 親OUに適用したSCPは、子OUやアカウントに継承される。
* **NotActionの活用**: IAMやRoute53など、グローバルサービスを制限から除外するために`NotAction`を適切に使用する。

## 🏢 ケースステディ

### ケース：開発者によるリージョン外でのリソース乱立

ある企業で、開発者がコスト削減を無視して、管理対象外の海外リージョンに高額なインスタンスを多数作成し、予期せぬ高額請求が発生しました。また、データ主権の観点から特定のリージョン以外でのデータ保持がコンプライアンス違反となるリスクが浮上しました。

#### 問題点

* リージョン利用に対する統制が効いていなかった。
* 開発者がどのリージョンを使用しているか可視化されていなかった。
* 誤操作や意図しないリージョン利用を防ぐガードレールが存在しなかった。

#### 対応策

* SCPによるリージョン制限を適用し、許可されたリージョン以外へのリソース作成をブロックする。
* AWS Configを使用して、非許可リージョンのリソースを検知・通知する仕組みを構築する。
* AWS Budgetsを設定し、予算超過時にアラートを飛ばす。
* 開発者への教育を実施し、適切なリージョン利用ガイドラインを策定する。

#### ケースから学ぶ教訓

* 信頼に基づいた管理ではなく、システム的なガードレール（SCP）による強制力が必要である。
* 「制限」と「可視化」の両輪でガバナンスを効かせる必要がある。
* セキュリティとコスト管理は密接に関連している。

## 📋 今日のまとめ

* AWS OrganizationsとOU設計により、アカウントの論理的な分離とガバナンスが実現できる。
* SCPは権限の最大範囲を制限するガードレールであり、組織全体のセキュリティ基準を強制できる。
* `aws:RequestedRegion`などの条件句を活用することで、地理的なコンプライアンスを担保できる。
