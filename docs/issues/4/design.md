# Design: Issue #4 【整理】既存ディレクトリの整理・不要ディレクトリの特定

## Background
- `Docs` と `Docs-Study` を統合して `my-docs` を作成する際、不要なファイルを排除し、整理されたディレクトリ構成を維持する。
- ユーザーフィードバックに基づき、`Docs-Study` を最優先とし、`Docs` からは日記のみをアーカイブする。
- **重要**: 最終検証 (Issue #6) が完了するまで、移行元のリポジトリおよびその Pipeline は現状維持とし、削除は行わない。

## Goals
- `Docs-Study` の環境を `my-docs` で完全に再現し、継続利用可能な状態を構築する。
- `Docs` から必要な日記ファイルのみを抽出してアーカイブする。
- `my-docs` における最適なディレクトリ構成（マージ方針）を確定させる。

## Proposed Approach
1. **棚卸し分析 (Refined)**:
    - `Docs-Study`: 現行の構成（`prompts/`, `templates/`, `tools/` 等を含む）を全移行対象とする。
    - `Docs`: 日記（`dev-records/`）のみを移行対象とし、他は除外する。
2. **整理・マージ方針の策定**:
    - 既存の `my-docs/research/`, `security-news/` との内容の競合を回避し、マージする。
3. **アーカイブ構成の確定**:
    - `archive/docs-dev-records/` に `Docs` の過去データを集約する。
4. **ユーザーレビュー**:
    - 修正後の整理リストと構成案をユーザーへ提示し、合意を得る。

## Success Criteria
- [ ] 継続利用する `Docs-Study` 由来の移行対象がすべて特定されている。
- [ ] `Docs` からの日記のみのアーカイブ計画が策定されている。
- [ ] 整理方針（特に既存コンテンツとのマージ）についてユーザーの承認が得られている。
