# Design: Issue #4 【整理】既存ディレクトリの整理・不要ディレクトリの特定

## Background
- `Docs` と `Docs-Study` を統合して `my-docs` を作成する際、不要なファイルを排除し、整理されたディレクトリ構造を維持する。
- **重要**: 最終検証 (Issue #6) が完了するまで、移行元のリポジトリおよびその Pipeline は現状維持とし、削除は行わない。本 Issue は「何を・どのように整理するか」の計画策定に焦点を当てる。

## Goals
- 移行対象（`Docs`, `Docs-Study`）の中から、統合不要なディレクトリ・ファイルを特定する。
- `my-docs` における最適なディレクトリ構成案を作成し、ユーザーの承認を得る。

## Proposed Approach
1. **棚卸し分析**:
    - `Docs` および `Docs-Study` の各ディレクトリの内容を精査する。
    - 過去のプロジェクト、重複ファイル、一時的な作業ファイルをリストアップする。
2. **整理方針の策定**:
    - 「アーカイブ（特定フォルダへ移動）」、「削除（my-docs へは移行しない）」、「統合（my-docs の新構成へ配置）」のいずれかを決定する。
3. **新ディレクトリ構成案の作成**:
    - 既存の構造を整理し、ナレッジベースとして使いやすい構成を提案する。
    - （例: `knowledge/`, `reports/`, `templates/`, `tools/` 等の分類）
4. **ユーザーレビュー**:
    - 整理リストと構成案をユーザーへ提示し、合意を得る。

## Success Criteria
- [ ] 不要・終了済みディレクトリのリストが作成されている。
- [ ] `my-docs` への移行対象と、その配置（新構成案）が確定している。
- [ ] 整理方針についてユーザーの承認が得られている。
