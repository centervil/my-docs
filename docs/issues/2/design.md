# Design: Issue #2 【分析】移行元リポジトリ (Docs, Doc-Study, private-ops) のクローンと現状分析

## Background
- 統合および移管作業を安全かつ確実に行うため、移行元のリポジトリ（`Docs`, `Doc-Study`）および移管先のリポジトリ（`private-ops`）の現状を詳細に把握する。

## Goals
- 移行対象となるファイル・ディレクトリの全容を把握する。
- 移管対象となる Pipeline コードおよびその依存関係（環境変数、シークレット等）を特定する。
- `my-docs` への統合および `private-ops` への移管に向けた具体的なパス構成案の基礎資料とする。

## Proposed Approach
1. **リポジトリの取得**:
    - `centervil/Docs`, `centervil/Doc-Study`, `centervil/private-ops` を作業ディレクトリ外（または `.gitignore` された一時ディレクトリ）にクローンする。
2. **構造分析**:
    - 各リポジトリのディレクトリツリーを取得し、ドキュメント、スクリプト、設定ファイルを分類する。
3. **Pipeline 分析**:
    - `.github/workflows` やスクリプトファイルを精査し、ドキュメント自動生成に関わるロジックとその入出力を特定する。
4. **依存関係の抽出**:
    - Pipeline が動作するために必要な環境変数（APIキー等）や、連携している外部サービスをリストアップする。

## Success Criteria
- [ ] 3 つのリポジトリの最新コードにアクセスできている。
- [ ] 各リポジトリの構成（ドキュメント、スクリプト、設定ファイル等）がドキュメント化されている。
- [ ] Pipeline の実行フローと必要な環境変数が特定されている。
